# Магазин

> **Status**: In Design
> **Author**: User + Claude
> **Last Updated**: 2026-04-15
> **Implements Pillar**: Pillar 1 (Рост героя), Pillar 5 (Понятная глубина)

## Overview

Магазин — торговый интерфейс в городе, доступный через NPC-торговца. Позволяет игроку продавать предметы из инвентаря (Equipment, Consumable, Junk) и покупать расходники и снаряжение из ассортимента магазина. Ассортимент фиксирован в MVP (не случайный); обновляется раз в сессию при входе в город. Цена покупки = `sell_value × 3`; цена продажи = `sell_value` (из Item Database). Репутация из Quest System применяет скидку к ценам покупки. QuestItem не продаётся. Магазин — главный золотой сток игры.

## Player Fantasy

Вернулся из данжа с рюкзаком хлама и парой приличных вещей. Быстро продаёшь мусор — и вот у тебя уже есть золото. Смотришь на полки магазина: вот зелье здоровья, вот редкий нагрудник — но не хватает 80 монет. Значит, ещё один данж. Магазин — это не просто транзакция, это планирование следующего шага.

## Detailed Design

### Core Rules

**Продажа:**
1. Игрок может продать любой предмет из рюкзака кроме QuestItem; экипированные предметы продать нельзя (сначала снять)
2. Цена продажи = `item.sell_value` (из Item Database); золото зачисляется мгновенно, предмет исчезает из рюкзака
3. Продажа необратима — предмет не возвращается

**Покупка:**
4. Ассортимент магазина: ~15 позиций — 5 слотов расходников (зелья разных размеров) + 10 слотов снаряжения (Common/Uncommon по всем слотам)
5. Цена покупки = `sell_value × 3` до применения скидки
6. Скидка по репутации: KNOWN −5%, RESPECTED −10%, HONORED −15% от цены покупки
7. Покупка невозможна если: недостаточно золота / рюкзак полон
8. После покупки предмет появляется в рюкзаке; золото списывается

**Ассортимент:**
9. Ассортимент обновляется раз в сессию при первом входе в город; хранится в `WorldState.shop_stock[]`
10. Купленный предмет исчезает из ассортимента до следующего обновления (не дублируется)
11. Ассортимент в MVP — фиксированный пул шаблонов, не случайный

### States and Transitions

**Состояния магазина:**

| Состояние | Условие | Поведение |
|-----------|---------|-----------|
| `CLOSED` | Игрок не у NPC-торговца | Интерфейс недоступен |
| `OPEN` | Игрок открыл диалог с NPC-торговцем | Показывает ассортимент и инвентарь игрока |
| `REFRESHED` | Первый вход в город за сессию | Ассортимент обновлён из пула |

**Состояния позиции ассортимента:**

```
AVAILABLE → SOLD_OUT  : игрок купил предмет
SOLD_OUT → AVAILABLE  : обновление сессии (новый вход в город)
```

### Interactions with Other Systems

| Система | Направление | Интерфейс |
|---------|-------------|-----------|
| **Inventory** | ↔ bidirectional | Продажа: `Inventory.remove_item(item_id)` → зачисление золота; Покупка: `Inventory.add_item(item_id) → bool` |
| **Item Database** | upstream → | `ItemDatabase.get_item(item_id)` → sell_value, display_name, icon, rarity |
| **PlayerData** | ↔ bidirectional | `PlayerData.gold` — читает баланс, списывает/зачисляет |
| **Quest System / World State** | upstream → | `WorldState.reputation[loc_id]` → вычисление скидки |
| **World State** | ↔ bidirectional | `WorldState.shop_stock[]` — хранит текущий ассортимент между визитами |
| **NPC System** | upstream → | ⚠️ *провизорно*: NPC открывает магазин через `ShopUI.open(npc_id)` |
| **SaveSystem** | downstream → | `WorldState` персистируется → ассортимент сохраняется |

## Formulas

- **Цена покупки**: `buy_price = sell_value × 3 × (1 − reputation_discount)`
  - `reputation_discount`: NEUTRAL = 0.0, KNOWN = 0.05, RESPECTED = 0.10, HONORED = 0.15
  - Пример: Common зелье HP (`sell_value = 20`) при RESPECTED → `20 × 3 × 0.9 = 54 золота`

- **Цена продажи**: `sell_price = item.sell_value` (без модификаторов)

- **Наценка магазина**: `markup = buy_price / sell_price = 3.0` (при NEUTRAL репутации)
  - При HONORED: `markup = 3.0 × 0.85 ≈ 2.55` — всё ещё прибыльно для магазина

- **Размер ассортимента**: 15 позиций = 5 Consumable + 10 Equipment (Common/Uncommon)

## Edge Cases

1. Покупка при недостатке золота → кнопка «Купить» неактивна; цена подсвечивается красным
2. Покупка при полном рюкзаке (20/20) → `Inventory.add_item()` возвращает `false`; сообщение «Нет места в рюкзаке»; золото не списывается
3. Продажа экипированного предмета → кнопка «Продать» неактивна; подсказка «Сначала снимите предмет»
4. Продажа QuestItem → кнопка «Продать» неактивна; предмет не исчезает
5. Ассортимент полностью раскуплен → UI показывает пустые слоты с «Нет в наличии»; обновится при следующей сессии
6. Смена репутации во время открытого магазина → цены пересчитываются немедленно (сигнал от WorldState)
7. Продажа стака Consumable → продаётся весь стак целиком; `sell_price = sell_value × stack_count`

## Dependencies

| Система | Тип зависимости | Интерфейс |
|---------|----------------|-----------|
| **Inventory** | Hard — без инвентаря нет источника предметов для продажи и места для купленного | `remove_item()`, `add_item()`, `get_all()` |
| **Item Database** | Hard — без данных предмета нет цен и отображения | `get_item(item_id)` |
| **PlayerData** | Hard — без золота нет транзакций | `PlayerData.gold` |
| **World State** | Hard — хранит ассортимент и репутацию | `shop_stock[]`, `reputation[]` |
| **NPC System** | Soft — ⚠️ провизорно; магазин открывается через NPC | `ShopUI.open(npc_id)` |
| **Quest System** | Soft — репутация влияет на цены, но магазин работает без неё | через `WorldState.reputation[]` |

## Tuning Knobs

| Knob | Текущее | Безопасный диапазон | Эффект при изменении |
|------|---------|--------------------|--------------------|
| `SHOP_MARKUP` | 3.0 | 2.0–5.0 | Ниже = доступнее покупка; выше = золото нужнее |
| `STOCK_SIZE` | 15 | 10–25 | Больше = разнообразнее выбор |
| `STOCK_REFRESH` | раз в сессию | per-visit / per-dungeon / never | Частота обновления ассортимента |
| `reputation_discount` | 5/10/15% | 3–10% за уровень | Ценность прокачки репутации |
| `sell_value` (по редкости) | 10–500 | ×0.5–×2.0 | Скорость накопления золота от продаж |

## Acceptance Criteria

1. Открытие магазина через NPC → UI показывает ассортимент (15 позиций) и инвентарь игрока
2. Продажа Junk предмета → золото зачислено (`PlayerData.gold` увеличился на `sell_value`); предмет исчез из рюкзака
3. Покупка при достаточном золоте и свободном рюкзаке → предмет в рюкзаке; золото списано; позиция ассортимента исчезла
4. Покупка при нехватке золота → кнопка неактивна; золото не изменилось
5. Покупка при полном рюкзаке → `add_item()` отклонён; золото не изменилось; сообщение в UI
6. Продажа QuestItem → кнопка неактивна; предмет остался
7. Репутация RESPECTED → цены покупки отображаются со скидкой −10%
8. Новая сессия (повторный вход в город) → ассортимент обновлён; ранее купленные позиции вернулись
