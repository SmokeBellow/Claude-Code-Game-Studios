# AI врагов (Enemy AI)

> **Status**: Approved
> **Author**: User + Claude Code Game Studios
> **Last Updated**: 2026-03-25
> **Implements Pillar**: Умная победа (Pillar 3), Героика с весом (Pillar 4)

## Overview

Система AI врагов управляет поведением всех боевых противников в данжах: каждый враг — отдельный `CharacterBody2D` с конфигурационным ресурсом (`EnemyData .tres`), конечным автоматом состояний (PATROL → CHASE → ATTACK) и `NavigationAgent2D` для обхода препятствий. MVP включает три архетипа: **Ближний боец** (агрессивный, melee), **Дальний боец** (дистанционный, отступает при сближении) и **Элитный** (комбинирует паттерны, применяет тактические паузы). Каждый враг использует Health & Stats для своих характеристик, испускает сигнал `enemy_attack_started(attacker, damage)` для детекции парирования Combat System и уничтожается через стандартный сигнал `died` при обнулении HP. Без этой системы данжи пусты — некому создавать угрозу, негде применять механики боя и нет «умной победы».

## Player Fantasy

Каждый тип врага — это вопрос с правильным ответом. **Ближний боец** давит, закрывает дистанцию и наказывает пассивность — он учит парировать и двигаться. **Дальний боец** заставляет думать о позиции: оставаться далеко — попадаешь под обстрел, подходить близко — он отступает и перегруппируется. **Элитный враг** сочетает оба давления и добавляет телеграфированные «особые» атаки — победа над ним ощущается как маленький мастер-класс. Вместе они создают то, что игрок после гибели думает «я знаю что сделал не так» — а не «это нечестно». Враги не читерят: их поведение предсказуемо в третий раз и смертоносно в первый.

## Detailed Design

### Core Rules

**Общая архитектура врага**

1. Каждый враг — `CharacterBody2D` с узлами: `NavigationAgent2D`, `DetectionArea` (Area2D), `HurtBox` (Area2D), `AnimatedSprite2D`, скрипт `BaseEnemy.gd`.
2. Параметры хранятся в `EnemyData.tres`: `display_name`, `max_hp`, `move_speed`, `attack_damage`, `attack_range`, `attack_cooldown`, `aggro_range`, `aggro_angle`, `patrol_radius`, `enemy_type`.
3. Масштабирование при спавне: `final_stat = base_stat * smoothing_multiplier(dungeon_level)` *(формула из Health & Stats GDD)*.
4. Движение: `NavigationAgent2D.set_target_position()` + `move_and_slide()` каждый кадр.

**Обнаружение игрока (Aggro)**

5. `DetectionArea` — Area2D с `CircleShape2D` радиуса `aggro_range`. Aggro срабатывает при (OR, код-проверка каждый кадр):
   - Игрок в DetectionArea И угол до игрока ≤ `aggro_angle / 2` (vision cone)
   - Враг получил урон от игрока (сигнал `health_changed` с отрицательным delta)
6. После aggro: `target` = игрок, переход в CHASE. Aggro не снимается до смерти врага или игрока (MVP).

**Архетип 1 — Ближний боец (Melee)**

| Параметр | Значение |
|----------|---------|
| HP | 50 |
| Урон | 12 |
| Скорость | 90 px/s |
| Радиус атаки | 60 px |
| Cooldown атаки | 1.0 с |
| Aggro range | 250 px |
| Aggro angle | 90° |
| Patrol radius | 150 px |

7. В CHASE: преследует игрока через NavigationAgent2D.
8. При `distance ≤ attack_range`: 0.4с анимация замаха (ATTACK_WINDUP) → испускает `enemy_attack_started(self, attack_damage)` → 1.0с cooldown → снова CHASE.

**Архетип 2 — Дальний боец (Ranged)**

| Параметр | Значение |
|----------|---------|
| HP | 35 |
| Урон снаряда | 8 |
| Скорость | 80 px/s |
| Preferred range | 200 px |
| Retreat threshold | 130 px |
| Max attack range | 350 px |
| Cooldown атаки | 1.5 с |
| Aggro range | 350 px |
| Aggro angle | 100° |

9. Если `distance < 130px` → REPOSITION (отступает от игрока). Если `distance > 350px` → ADVANCE (приближается). Иначе — стоит и атакует.
10. При атаке: 0.4с анимация прицеливания → создаёт `EnemyProjectile` в направлении игрока → 1.5с cooldown. Снаряд: скорость 200 px/с, время жизни 2с.

**Архетип 3 — Элитный враг (Elite)**

| Параметр | Значение |
|----------|---------|
| HP | 100 |
| Урон melee | 18 |
| Урон рывка | 22 |
| Скорость | 100 px/s |
| Скорость рывка | ×3 (300 px/s) |
| Melee range | 70 px |
| Charge range | 200 px |
| Cooldown | 0.5с пауза после любой атаки |

11. Выбор атаки: если `distance ≤ 70px` → melee (0.4с телеграф); если `70px < distance ≤ 200px` → рывок (0.6с телеграф + индикатор направления); если `distance > 200px` → CHASE.
12. После каждой атаки: 0.5с тактическая пауза (стоит, поворачивается к игроку).
13. Рывок прерывается при коллизии со стеной — враг останавливается, cooldown запускается.

**Patrol**

14. В PATROL: выбирает случайную точку в радиусе `patrol_radius` от стартовой позиции → идёт → ждёт `rand(1.0, 3.0)` с → следующая точка.

### States and Transitions

| Состояние | Кто имеет | Условие входа | Условие выхода |
|-----------|-----------|--------------|----------------|
| **PATROL** | Все | Старт *(MVP: aggro не снимается — враг остаётся в CHASE до смерти)* | Aggro сработал |
| **CHASE** | Melee, Elite | Aggro | `distance ≤ attack_range` |
| **REPOSITION** | Ranged | `distance < retreat_range` | `distance ≥ preferred_range` |
| **ADVANCE** | Ranged | `distance > max_range` | `distance ≤ max_range` → переоценка условий каждый кадр: если 130px ≤ distance ≤ 350px — ATTACK_WINDUP; если distance < 130px — REPOSITION |
| **ATTACK_WINDUP** | Все | В зоне атаки + cooldown истёк | Windup таймер истёк *(Elite: 0.4с для melee, 0.6с для charge — определяется выбранной атакой)* |
| **ATTACK** | Все | После WINDUP | Удар нанесён / снаряд выпущен |
| **ATTACK_COOLDOWN** | Все | После ATTACK | cooldown таймер истёк |
| **DEAD** | Все | HP ≤ 0 | — (финальное) |

*В состоянии DEAD: коллизии отключены, навигация остановлена, воспроизводится анимация смерти, через 0.5с враг удаляется из дерева.*

### Interactions with Other Systems

| Система | Направление | Данные |
|---------|-------------|--------|
| **Health & Stats** | ← Enemy AI | Читает `max_hp`, `move_speed`, `attack_damage` через `EnemyData` + smoothing_multiplier |
| **Health & Stats** | → Enemy AI | Испускает `died` → переход в DEAD; `health_changed(delta)` → aggro при отрицательном delta |
| **Player Movement** | ← Enemy AI | Читает `player.global_position` для расчёта дистанции и направления |
| **Combat System** | → Enemy AI | Наносит урон при попадании атаки в `HurtBox` врага |
| **Combat System** | ← Enemy AI | Слушает `enemy_attack_started(attacker, damage)` для детекции парирования |
| **NavigationAgent2D** | ← Enemy AI | Вызывает `set_target_position()` каждые 0.1с (throttled) |

## Formulas

```
# Масштабирование статов при спавне
final_stat = base_stat * smoothing_multiplier(dungeon_level)
# smoothing_multiplier — из Health & Stats GDD, применяется одинаково к врагам и герою

# Урон ближнего удара и снаряда
enemy_melee_damage = attack_damage          # flat, из EnemyData.tres
enemy_projectile_damage = attack_damage     # тот же параметр для Ranged
elite_charge_damage = charge_damage         # отдельное поле в EnemyData (Elite)

# Проверка vision cone
direction_to_player = (player.global_position - global_position).normalized()
angle_to_player = facing_direction.angle_to(direction_to_player)
is_in_cone = abs(angle_to_player) <= aggro_angle / 2.0

# Дистанция до игрока
distance = global_position.distance_to(player.global_position)

# Скорость рывка Elite
charge_speed = move_speed * 3.0  # 100 * 3 = 300 px/s

# Throttle обновления NavigationAgent2D
nav_update_timer += delta
if nav_update_timer >= 0.1:
    navigation_agent.set_target_position(player.global_position)
    nav_update_timer = 0.0
```

| Переменная | Источник | Значение ур.1 |
|------------|----------|--------------|
| `base_stat` | EnemyData.tres | см. таблицы архетипов |
| `smoothing_multiplier` | Health & Stats GDD | 1.0 на ур.1 |
| `aggro_angle` | EnemyData.tres | 90° (Melee), 100° (Ranged) |
| `charge_damage` | EnemyData.tres (Elite) | 22 |

## Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| Игрок убивает врага в момент ATTACK_WINDUP | Враг переходит в DEAD, `enemy_attack_started` не испускается — урон не наносится |
| Несколько врагов атакуют одновременно | Каждый работает независимо; парирование (Combat System) обрабатывает только первый удар в окне — остальные наносят урон |
| NavigationAgent2D не может найти путь (враг за стеной) | Враг остаётся в CHASE, пытается каждые 0.1с; если путь не найден 3с подряд — прямолинейное движение как fallback |
| Снаряд (Ranged) попадает в другого врага | Снаряд уничтожается, урон другому врагу не наносится (MVP: friendly fire отключён) |
| Элитный рывок: стена между врагом и игроком | Враг разгоняется и останавливается при коллизии через `move_and_slide()`; cooldown запускается немедленно |
| Aggro через стену (sound aggro) | Не реализован в MVP — только vision cone и on-hit aggro |
| Patrol: не может достичь случайной точки (тупик) | Через 3с таймаут — выбирает новую случайную точку |
| Враг умер, снаряд в воздухе | Снаряд продолжает полёт и наносит урон при попадании (снаряд — независимая сцена) |
| HP = 1, получает точный урон до 0 и одновременно атакует | DEAD имеет приоритет; если `died` испущен до `enemy_attack_started` — атака отменяется |
| Игрок быстро меняет позицию | Путь пересчитывается каждые 0.1с — задержка максимум 0.1с; приемлемо для MVP |

## Dependencies

**Входящие (Enemy AI зависит от):**

| Система | Что берёт | Тип |
|---------|-----------|-----|
| **Health & Stats** | `max_hp`, `smoothing_multiplier(level)`; получает сигналы `died`, `health_changed` | Жёсткая |
| **Player Movement** | `player.global_position` для детекции и навигации | Жёсткая |
| **NavigationAgent2D** | Встроенный Godot-узел — pathfinding по NavigationRegion2D уровня | Жёсткая |

**Исходящие (зависят от Enemy AI):**

| Система | Что берёт |
|---------|-----------|
| **Combat System** | Сигнал `enemy_attack_started(attacker, damage)` для детекции парирования; `HurtBox` Area2D для получения урона |
| **Система боссов (#23)** | Использует те же архетипы и `BaseEnemy.gd` как базовый класс |
| **Лут-система (#12)** | Слушает сигнал `enemy_died(xp_reward: int, enemy_data: EnemyData)` — читает `enemy_data.loot_table_id` для выбора таблицы дропа |
| **Система уровней и XP (#8)** | Слушает сигнал `enemy_died(xp_reward: int, enemy_data: EnemyData)` — использует `xp_reward` для начисления XP |

## Tuning Knobs

| Параметр | Текущее | Безопасный диапазон | Что ломается при выходе |
|----------|---------|---------------------|------------------------|
| HP Melee | 50 | 30–80 | <30: убивается 2-3 ударами, нет времени на парирование; >80: бой затягивается |
| Урон Melee | 12 | 8–20 | <8: нет угрозы; >20: 2 удара убивают игрока ур.1 (HP=80) |
| HP Ranged | 35 | 20–55 | <20: убивается с одного удара; >55: Ranged переживает несколько атак — слишком живучий для дистанционника |
| Урон Ranged снаряда | 8 | 5–15 | <5: нет угрозы от дистанции; >15: превышает Melee, несбалансировано |
| HP Elite | 100 | 70–150 | <70: элитник умирает как обычный враг; >150: бой затягивается, игрок тратит ресурсы |
| Windup телеграфа (base) | 0.4с | 0.2–0.7с | <0.2с: слишком быстро для реакции; >0.7с: парирование тривиально |
| Windup Elite рывка | 0.6с | 0.4–1.0с | <0.4с: невозможно среагировать; >1.0с: слишком долгое ожидание |
| Aggro range Melee | 250 px | 150–350 px | <150: враги замечают игрока слишком поздно; >350: aggro через всю комнату |
| Aggro range Ranged | 350 px | 250–450 px | <250: Ranged обнаруживает игрока когда тот уже рядом; >450: вне разумного предела комнаты |
| Retreat threshold Ranged | 130 px | 80–180 px | <80: Ranged подпускает вплотную; >180: не даёт игроку сближаться |
| Elite charge range | 200 px | 120–280 px | <120: рывок применяется редко; >280: практически всегда использует рывок |
| Elite charge speed | ×3 | ×2–×4 | <×2: рывок слабее ожиданий; >×4: невозможно уклониться |
| NavAgent2D throttle | 0.1с | 0.05–0.2с | <0.05с: лишняя нагрузка при многих врагах; >0.2с: заметное запаздывание реакции |
| Patrol wait time | rand(1–3)с | 0.5–5с | <0.5с: враги нервозно мечутся; >5с: кажутся статичными |

## Visual/Audio Requirements

**Визуальные эффекты:**
- **Ближний боец**: анимация замаха с лёгким motion blur на оружии; при ударе — короткий screen shake (0.05с)
- **Дальний боец**: анимация прицеливания (конус перед выстрелом, 0.3с); снаряд — светящийся projectile с trail-эффектом
- **Элитный**: перед рывком — индикатор стрелки направления (красная полоса в world space, 0.6с — весь ATTACK_WINDUP); при рывке — trail за врагом
- **Aggro**: знак «!» над врагом на 0.3с при входе в CHASE
- **DEAD**: fade-out 0.3с + удаление сцены

**Звук:**
- **Melee замах**: свист оружия (~0.15с до удара)
- **Melee удар**: тяжёлый глухой удар; ×3 вариации (рандом)
- **Ranged выстрел**: звук броска; звук попадания снаряда отдельно
- **Elite рывок**: нарастающий звук ускорения + тяжёлый удар при коллизии
- **Aggro**: короткий grunt/рычание при обнаружении игрока
- **Получение урона**: grunt боли ×3 вариации
- **Смерть**: отдельный звук, отличается от grunt урона

## UI Requirements

- MVP: нет health bar над обычными врагами — угрозу читают по анимации, не по цифрам
- **Элитный враг**: тонкая health bar над спрайтом (только для него, как мини-босс)
- Телеграф рывка Elite — индикатор стрелки в world space (не HUD), исчезает после ATTACK_WINDUP

## Acceptance Criteria

| # | Критерий | Как проверить |
|---|----------|--------------|
| 1 | Ближний боец патрулирует до aggro: ходит по случайным точкам в радиусе `patrol_radius` | Поставить врага в пустую комнату без игрока, наблюдать движение |
| 2 | Aggro через vision cone: враг не замечает игрока сзади | Подойти к Melee сзади вне cone — не реагирует; зайти спереди — aggro срабатывает |
| 3 | Aggro on-hit: враг реагирует на удар независимо от направления взгляда | Ударить Ranged сзади — переходит в CHASE |
| 4 | Ближний боец преследует через NavigationAgent2D, огибает стены | Поставить стену между врагом и игроком — враг обходит препятствие |
| 5 | Телеграф 0.4с перед ударом Melee создаёт окно парирования | Засечь время от начала анимации до `enemy_attack_started` — 0.35–0.45с |
| 6 | Дальний боец отступает при `distance < 130px` | Подойти вплотную — Ranged начинает отходить от игрока |
| 7 | Снаряд Ranged летит 200px/с и исчезает через 2с | В пустой комнате снаряд пропадает ~400px от точки выстрела |
| 8 | Элитный рывок телеграфируется 0.6с и прерывается стеной | Встать за стеной от Elite — рывок начинается, останавливается при коллизии |
| 9 | Все враги умирают через сигнал `died` и удаляются из дерева | Снизить HP до 0 в отладчике — сцена удаляется через 0.5с |
| 10 | `enemy_attack_started` отлавливается Combat System для парирования | Логирование сигнала: парирование (ПКМ) в нужный момент блокирует урон |
| 11 | 5+ Melee + 3 Ranged + 1 Elite одновременно — без просадок FPS | Заспавнить 9 врагов в одной комнате — FPS остаётся ≥50 |

## Open Questions

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | Нужен ли sound aggro (слышит атаки рядом)? | MVP: нет — только vision cone и on-hit |
| 2 | Элитный — уникальный арт или улучшенная версия Melee? | Уникальный цвет/размер спрайта + дополнительный эффект |
