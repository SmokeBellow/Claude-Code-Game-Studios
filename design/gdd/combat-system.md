# Боевая система (Combat System)

> **Status**: Approved
> **Author**: User + Claude Code Game Studios
> **Last Updated**: 2026-03-25
> **Implements Pillar**: Умная победа (Pillar 3), Рост героя (Pillar 1)

## Overview

Боевая система управляет всеми взаимодействиями «герой против врагов» в реальном времени: базовая атака с хитбоксом, класс-специфичная защитная механика (Воин — парирование, Ловкач — уворот, Маг — щит), активные способности с кулдаунами и расчёт урона с учётом критов и уклонений. Система читает статы из Health & Stats, блокирует движение через Player Movement на время атаки и управляет жизненным циклом боевых событий — от нажатия кнопки до применения урона. Без боевой системы невозможны ни проверка пиллера «умная победа», ни ощущение роста героя от лута и прокачки.

## Player Fantasy

Бой — это баланс между напором и умом. Каждый класс ощущается принципиально по-своему: Воин не уворачивается — он держит удар, читает момент и бьёт в ответ с удвоенной силой. Ловкач скользит между ударами и наказывает врага за промах. Маг контролирует пространство и щитом превращает атаку врага против него самого. Игрок, который понял паттерн врага, всегда побеждает элегантнее того, кто просто закликивает. К концу сессии он думает не «я нажал кнопку» — он думает «я перехитрил этого врага».

## Detailed Design

### Core Rules

**Управление (Воин)**

1. **LMB** — базовая атака: мгновенный хитбокс в направлении `facing_direction`, блокирует движение на время анимации (~0.4с). Cooldown: 0.4с.
2. **ПКМ** — парирование: Воин входит в стойку на 0.35с (окно парирования). Во время окна Воин неподвижен.
   - Если враг атаковал в окне → **контрудар**: следующая LMB-атака в течение 0.5с наносит ×2 урон
   - Если никто не атаковал → **уязвимость**: Воин теряет 0.4с без атаки и защиты
   - Cooldown парирования: 1.5с
3. **Q** — «Рывок»: Воин совершает анимированный прыжок (0.2с) к ближайшему врагу в радиусе 200px. Движение и все действия заблокированы во время полёта (нет i-фреймов). При приземлении наносит ×1.5 физического урона. После приземления — 0.5с блокировка ПКМ (парирование недоступно). Стоимость: **20 маны**. Cooldown: 5с.
4. **E** — «Вихрь»: Воин вращается 2с, нанося урон всем врагам в радиусе 60px каждые 0.3с, и получает +30% к скорости перемещения на время вращения. Движение во время вращения разрешено. Стоимость: **35 маны**. Cooldown: 8с.

**Расчёт урона (общие правила)**

4. Уклонение проверяется первым — если сработало, урон не применяется.
5. Затем проверяется крит: `final_damage = (weapon_base + phys_bonus) * crit_multiplier` (если крит) или `weapon_base + phys_bonus` (обычный).
6. Контрудар умножает итоговый урон на ×2 после всех расчётов.
7. Урон противника не может снизить HP ниже 0 — `hp = max(0, hp - damage)`.

### States and Transitions

| Состояние | Условие входа | Условие выхода | Что заблокировано |
|-----------|--------------|----------------|-------------------|
| **Idle** | Старт / после любого действия | LMB / ПКМ / E нажата | — |
| **Атака** | LMB нажата | Анимация завершена (~0.4с) | Движение, атака, парирование |
| **Парирование** | ПКМ нажата | Истекло 0.35с или сработал контрудар | Движение, атака, E |
| **Уязвимость** | Парирование без попадания | Истекло 0.4с | Атака, парирование, E |
| **Рывок (Q)** | Q нажата, есть враг в 200px, есть мана | Приземление (0.2с) | Движение, атака, парирование, E |
| **После рывка** | Приземление | Истекло 0.5с | Парирование |
| **Вихрь (E)** | E нажата, есть мана | Истекло 2с | Атака, парирование |
| **Контрудар** | Парирование сработало + LMB в 0.5с | Удар нанесён | — |
| **Мёртв** | HP = 0 | Перезагрузка сцены | Всё |

### Interactions with Other Systems

| Система | Направление | Данные |
|---------|-------------|--------|
| **Health & Stats** | ← Combat | Читает: HP, `phys_damage_bonus`, `dodge_chance`, `crit_chance`, `crit_multiplier`, `atk_speed`, `move_speed` |
| **Health & Stats** | → Combat | Вызывает `take_damage(amount)` и `heal(amount)` на враге/герое |
| **Player Movement** | → Combat | Combat вызывает `set_movement_locked(true/false)` (`is_movement_locked` флаг в Player Movement) во время атаки, парирования, уязвимости |
| **Player Movement** | ← Combat | Читает `global_position`, `facing_direction` для позиционирования хитбоксов |
| **Система способностей** | ← Combat | Регистрирует Вихрь (E) как способность; Combat вызывает `spend_mana(amount)` через неё |
| **AI врагов** | ← Combat | Враги используют тот же расчёт урона; Combat слушает сигнал `enemy_attack_started(attacker: Node, damage: float)` для проверки окна парирования |
| **Боевой фидбек** | → Combat | Слушает события: `attack_hit`, `parry_success`, `parry_failed`, `counter_hit`, `enemy_died` — для VFX/SFX |
| **HUD** | → Combat | Слушает состояния: парирование (индикатор окна), уязвимость (вспышка), контрудар (иконка) |

## Formulas

```
# Базовый урон
final_damage = weapon_base + phys_damage_bonus                               # обычный удар
final_damage = (weapon_base + phys_damage_bonus) * crit_multiplier           # крит

# Контрудар
counter_damage = final_damage * 2.0

# Хитбокс LMB — полукруг 80px, дуга 110° в направлении facing_direction

# Вихрь — урон за тик (7 тиков за 2с → суммарно ~280% от базового удара)
# Стоимость: 35 маны
whirl_tick_damage = (weapon_base + phys_damage_bonus) * 0.4

# Cooldown с учётом скорости атаки (из Health & Stats)
actual_cooldown = base_cooldown / atk_speed

# Проверка уклонения (до урона)
roll = randf()  # 0.0–1.0
if roll < dodge_chance:   # dodge_chance из Health & Stats (0.05–0.40)
    pass  # урон не применяется

# Проверка крита (после уклонения)
roll = randf()
if roll < crit_chance:    # crit_chance из Health & Stats (0.05–0.50)
    apply crit_multiplier
```

| Переменная | Источник | Значение на ур.1 |
|------------|----------|-----------------|
| `weapon_base` | Item Database | ~25 |
| `phys_damage_bonus` | Health & Stats (STR) | +5 |
| `crit_multiplier` | Health & Stats (LCK) | 1.5× |
| `crit_chance` | Health & Stats (LCK) | 5% |
| `dodge_chance` | Health & Stats (DEX) | 5% |
| `atk_speed` | Health & Stats (DEX) | 1.0× |

*Формулы `final_damage` и `actual_cooldown` согласованы с `design/gdd/health-stats.md`, секция Formulas — не дублируем, ссылаемся.*

## Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| LMB во время Вихря | Игнорируется — атака заблокирована в состоянии Вихрь |
| ПКМ во время Вихря | Игнорируется — парирование заблокировано |
| E нажата без маны | Вихрь не активируется, мана не тратится |
| Враг атакует в момент входа в уязвимость | Уязвимость применяется — урон полный |
| Контрудар не использован за 0.5с | Окно закрывается, обычная атака возобновляется |
| Два врага атакуют одновременно в окне парирования | Оба засчитываются — одно парирование, контрудар наносится по ближайшему врагу в `facing_direction` |
| Смерть во время Вихря | Вихрь прерывается, переход в состояние «Мёртв» |
| LMB нажата во время cooldown | Игнорируется — атака не начинается раньше cooldown |
| Враг убит в окне парирования | Контрудар не применяется — цели нет |
| Вихрь задевает 0 врагов | Мана потрачена, cooldown запущен — никакого возврата |

## Tuning Knobs

| Параметр | Текущее значение | Безопасный диапазон | Что ломается при выходе |
|----------|-----------------|---------------------|-------------------------|
| Cooldown базовой атаки | 0.4с | 0.2–0.7с | <0.2с: закликивание без тактики; >0.7с: атака ощущается вялой |
| Окно парирования | 0.35с | 0.2–0.5с | <0.2с: Dark Souls — слишком жёстко для MVP; >0.5с: парирование тривиально |
| Cooldown парирования | 1.5с | 1.0–2.5с | <1.0с: спам парирования убивает риск; >2.5с: игрок боится использовать |
| Длительность уязвимости | 0.4с | 0.2–0.8с | <0.2с: наказание не ощущается; >0.8с: слишком жестокое наказание |
| Окно контрудара | 0.5с | 0.3–0.8с | <0.3с: сложно успеть нажать LMB; >0.8с: контрудар теряет напряжение |
| Множитель контрудара | ×2.0 | ×1.5–×3.0 | <×1.5: награда не ощущается; >×3.0: мгновенное убийство большинства врагов |
| Длительность Вихря | 2.0с | 1.5–3.0с | <1.5с: не успеваешь войти в толпу; >3.0с: слишком долго быть уязвимым |
| Cooldown Вихря | 8с | 5–15с | <5с: Вихрь становится основной атакой; >15с: игрок забывает о способности |
| Урон Вихря за тик | 40% | 25–60% | <25%: не стоит риска входить в толпу; >60%: Вихрь лучше базовой атаки |
| Стоимость маны Вихря | 35 | 20–50 | <20: слишком дёшево, без ресурс-менеджмента; >50: нельзя использовать на ур.1 |
| Радиус Вихря | 60px | 40–90px | <40px: сложно задеть больше 1 врага; >90px: задевает врагов через стены |
| Бонус скорости в Вихре | +30% | +15–+50% | <15%: бафф не ощущается; >50%: Вихрь становится лучшим инструментом побега |
| Размер хитбокса LMB | 80px / 110° | 60–100px / 90–130° | <60px: сложно попасть; >100px: атака задевает врагов сбоку, теряет тактику |

## Visual/Audio Requirements

**Визуальные эффекты:**
- **Атака**: короткая вспышка на хитбоксе (0.1с) + белый flash на враге при попадании
- **Парирование**: синий ореол вокруг Воина на время окна (0.35с)
- **Контрудар**: золотой flash на Воине + увеличенные числа урона (другой цвет)
- **Уязвимость**: красный flash на Воине
- **Вихрь**: частицы вращения вокруг Воина, радиус визуально обозначен

**Звук:**
- **Атака**: звук удара мечом (тяжёлый, металлический)
- **Попадание по врагу**: звук удара по плоти / брони
- **Парирование активно**: тихий металлический звон (вход в стойку)
- **Контрудар**: мощный звук удара, отличный от обычного
- **Промах парирования**: звук уязвимости (свист воздуха или "провал")
- **Вихрь**: свист вращающегося оружия, нарастающий в начале

## UI Requirements

**HUD во время боя:**
- **Индикатор парирования**: синяя рамка / иконка щита появляется на 0.35с при активном окне парирования
- **Индикатор контрудара**: золотая иконка меча мигает 0.5с — напоминание нажать LMB
- **Индикатор уязвимости**: красная вспышка по краям экрана (виньет) на 0.4с
- **Cooldown E**: иконка Вихря с таймером обратного отсчёта (8с)
- **Cooldown парирования**: иконка щита с таймером (1.5с)

**Числа урона (DamageNumber):**
- Обычный удар: белые числа, всплывают вверх
- Крит: жёлтые числа, чуть крупнее
- Контрудар: золотые числа с восклицательным знаком, заметно крупнее
- Тики Вихря: серые числа, мелкие (не перегружать экран)

## Acceptance Criteria

| # | Критерий | Как проверить |
|---|----------|---------------|
| 1 | LMB наносит урон врагу в `facing_direction`, движение блокируется на ~0.4с | Атаковать стоя, наблюдать блокировку движения и урон на враге |
| 2 | ПКМ открывает окно парирования 0.35с — визуальный индикатор в HUD виден | Нажать ПКМ, наблюдать индикатор в HUD |
| 3 | Враг атакует в окне → следующая LMB наносит ×2 урон | Намеренно поймать атаку врага, проверить числа урона |
| 4 | ПКМ без атаки врага → 0.4с уязвимости без атаки и защиты | Нажать ПКМ вхолостую, попытаться атаковать сразу после |
| 5 | Q активирует Рывок: Воин прыгает к ближайшему врагу в 200px, наносит ×1.5 урона при приземлении | Встать рядом с врагом, нажать Q, проверить позицию и урон |
| 6 | После Рывка ПКМ заблокировано на 0.5с | Нажать Q, сразу нажать ПКМ — парирование не активируется |
| 7 | E активирует Вихрь: Воин движется, наносит тиковый урон каждые 0.3с, +30% скорость | Активировать E рядом с врагами, наблюдать тики урона и скорость |
| 8 | E не активируется при нехватке маны | Истратить ману, нажать E — ничего не происходит |
| 9 | Контрудар недоступен если LMB не нажата в течение 0.5с после парирования | Поймать атаку, подождать 0.6с — окно должно закрыться |
| 10 | Уклонение из DEX проверяется перед уроном — при уклонении урона нет | Поднять DEX до 40% уклонения, наблюдать промахи врагов |
| 11 | Смерть во время Вихря прерывает Вихрь и блокирует всё управление | Получить летальный урон во время E |
| 12 | Все события (`attack_hit`, `parry_success` и т.д.) испускаются и принимаются HUD/фидбеком | Проверить через отладчик сигналов Godot |

## Open Questions

Все вопросы разрешены в ходе проектирования:

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | Универсальная защита для всех классов или уникальная? | Уникальная — каждый класс имеет свою защитную механику без общего fallback |
| 2 | Какой класс первым в MVP? | Воин (Сила): парирование + контрудар + Вихрь |
| 3 | Что происходит при промахе парирования? | Уязвимость 0.4с — наказание за агрессивное использование |
| 4 | Крит + контрудар — не слишком мощно? | Принято как высокая награда за точный тайминг; для боссов можно ввести `counter_multiplier_vs_boss` при необходимости |
