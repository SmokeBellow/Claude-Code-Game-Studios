# Quest System

> **Status**: In Design
> **Author**: User + Claude
> **Last Updated**: 2026-04-15
> **Implements Pillar**: Pillar 2 (Живой мир), Pillar 3 (Умная победа)

## Overview

Quest System — система заданий, структурирующая нарратив и дающая игроку цель в каждой сессии. MVP реализует одну линейную цепочку от Старейшины (4 стадии: убить 5 врагов → Печать элиты → Страж Данжа → завершение). В полной версии система поддерживает несколько NPC-квест-гиверов с ветвящимися цепочками: выбор игрока влияет на доступные ветки и финальные награды. Квесты отслеживаются через централизованный `QuestSystem` (Autoload), состояние персистируется через SaveSystem.

## Player Fantasy

Игрок идёт в данж не просто убивать — у него есть конкретная цель от конкретного человека. Каждое возвращение в город — это не просто сохранение, а момент: NPC реагирует на сделанное, выдаёт следующий этап, мир немного меняется. Выполненный квест даёт три вещи одновременно: золото и прогресс (материально), ощущение что ты помог живому миру (эмоционально), и новый кусочек лора — зачем этот данж вообще существует.

## Detailed Design

### Core Rules

**Общие правила:**
1. Квест состоит из стадий; каждая стадия имеет: цель, условие выполнения, награду стадии
2. Ветвление: выбор в диалоге → открывает одну из веток следующей стадии (для сюжетных и классовых)
3. Не более 3 активных квестов одновременно (один сюжетный, один классовый, один репутационный + до 2 охотничьих в лимите)
4. Статусы квеста: `NOT_STARTED → ACTIVE → STAGE_READY → COMPLETED`
5. Принятие: игрок разговаривает с NPC / берёт с доски → квест становится `ACTIVE`
6. Завершение стадии: условие выполнено + возврат к NPC → награда + переход к следующей стадии
7. Квестовые предметы привязаны к `PlayerData.quest_items`, не к инвентарю — не теряются при смерти
8. Завершение квеста меняет диалог NPC навсегда (флаг в `WorldState`)
9. ⚠️ `QuestSystem.gd` отсутствует в main-ветке; логика разбросана по `PlayerData` и `ElderNPC` — требует централизации

**Типы квестов:**

| Тип | Константа | Источник | Повторяемость | Уникальная механика |
|-----|-----------|----------|---------------|-------------------|
| **Сюжетные** | `STORY` | Старейшина | Нет | Открывают следующий акт; разблокируют новые зоны данжа |
| **Классовые** | `CLASS` | NPC класса (новый) | Нет | Награда — умение или лор класса; доступны после выбора класса на ур. 3 |
| **Репутационные** | `REPUTATION` | Любой NPC в городе | Нет | Завершение даёт репутацию в локации/группе → бонусы у связанных NPC |
| **Охотничьи** | `BOUNTY` | Доска объявлений | Да (сбрасываются каждую сессию) | Нет ветвления; простая цель (убить N врагов / найти предмет); только золото |

**Правила по типам:**
- **Сюжетные**: линейная цепочка, блокирует прогресс данжа если не выполнен — единственный обязательный квест
- **Классовые**: доступны только если `PlayerData.player_class != NONE`; один активный классовый квест за раз
- **Репутационные**: `WorldState.reputation[location_id] += delta` при завершении; репутация влияет на цены в магазине и диалоги
- **Охотничьи**: генерируются из пула шаблонов при входе в сессию; не сохраняются между сессиями; можно взять до 2 штук одновременно

### States and Transitions

**Состояния квеста:**

| Состояние | Условие входа | Что видит игрок |
|-----------|--------------|----------------|
| `NOT_STARTED` | Квест существует в базе, игрок не принял | NPC предлагает квест в диалоге |
| `ACTIVE` | Игрок принял квест | Цель отображается в Quest Log |
| `STAGE_READY` | Условие стадии выполнено | Маркер на NPC / доске; в Quest Log — «Вернитесь к [NPC]» |
| `COMPLETED` | Награда получена, финальная стадия | NPC получает завершённый диалог навсегда |

**Переходы:**
```
NOT_STARTED → ACTIVE      : игрок принял квест в диалоге / взял с доски
ACTIVE → STAGE_READY      : условие стадии выполнено (убил N, нашёл предмет и т.д.)
STAGE_READY → ACTIVE      : игрок вернулся к NPC, получил награду стадии, открылась следующая стадия
STAGE_READY → COMPLETED   : игрок вернулся к NPC на финальной стадии
ACTIVE → NOT_STARTED      : игрок отказался (только охотничьи — через контекстное меню доски)
```

**Состояния репутации (только `REPUTATION`-квесты):**
```
NEUTRAL (0) → KNOWN (1–2 квеста) → RESPECTED (3–4) → HONORED (5+)
```
Каждый уровень даёт скидку у связанных NPC (+5% за уровень) и новые диалоговые опции.

### Interactions with Other Systems

| Система | Направление | Интерфейс |
|---------|-------------|-----------|
| **NPC System** | upstream → | NPC вызывает `QuestSystem.offer_quest(quest_id)` в диалоге; читает `is_quest_active(quest_id)` для смены диалога |
| **World State** | ↔ bidirectional | Флаги завершения: `WorldState.set_flag("quest_[id]_done")`; репутация: `WorldState.reputation[loc_id]` |
| **Dialogue System** | upstream → | Диалог читает `QuestSystem.get_status(quest_id)` → выбирает ветку диалога |
| **PlayerData / SaveSystem** | ↔ bidirectional | `active_quests[]`, `completed_quests[]`, `quest_items[]` — персистируются |
| **Enemy AI / Combat** | downstream → | `QuestSystem.notify_kill(enemy_type, count)` вызывается при смерти врага |
| **Inventory** | downstream → | Квестовые предметы не в инвентаре; в `PlayerData.quest_items` — не теряются |
| **Skill Tree** | downstream → | Классовые квесты: `ClassAbilitySystem.unlock_ability(ability_id)` при `COMPLETED` |
| **Shop** | downstream → | Магазин читает `WorldState.reputation[loc_id]` → применяет скидку |
| **Notice Board** (новый) | upstream → | UI-нода, генерирует пул охотничьих квестов при старте сессии |

## Formulas

- **Золото за квест**: `gold = base_reward × stage_multiplier × (1 + reputation_bonus)`
  - `base_reward`: Охотничьи = 50–150, Репутационные = 100–300, Классовые = 200, Сюжетные = 150–400
  - `stage_multiplier`: стадия 1 = 1.0, стадия 2 = 1.5, стадия 3 = 2.0
  - `reputation_bonus`: NEUTRAL = 0.0, KNOWN = 0.05, RESPECTED = 0.10, HONORED = 0.15

- **Репутация за квест**: `reputation_delta = base_rep × quest_difficulty`
  - `base_rep`: лёгкий = 1, средний = 2, сложный = 3
  - Пороги уровней: KNOWN ≥ 2, RESPECTED ≥ 5, HONORED ≥ 9 (сумма по всем квестам локации)

- **Пул охотничьих квестов**: `pool_size = 4` шаблона; выбираются случайно при старте сессии; игрок берёт ≤ 2

## Edge Cases

1. Игрок пытается взять 4-й квест при лимите → `QuestSystem.accept_quest()` возвращает `false`, диалог: «Закончи текущие задания»
2. Охотничий квест: игрок убил врагов до принятия квеста → счётчик не засчитывается (только после `ACTIVE`)
3. Смерть с квестовым предметом → предмет в `PlayerData.quest_items`, не теряется
4. NPC квест-гивер не умирает (флаг `immortal = true`) — квест не зависает в `STAGE_READY` навсегда
5. Классовый квест активен → игрок сбрасывает класс у Волшебника → квест отменяется, статус → `NOT_STARTED`, предметы возвращаются
6. Охотничьи квесты при новой сессии → сбрасываются; `ACTIVE` охотничьи при выходе из данжа → теряются (охотничьи — высокорисковые по дизайну)
7. Два квеста требуют один предмет → предмет общий; первый завершённый забирает; второй ждёт следующего экземпляра

## Dependencies

| Система | Тип зависимости | Интерфейс |
|---------|----------------|-----------|
| **NPC System** | Hard — без NPC нет источника квестов | `offer_quest()`, `get_status()` |
| **World State** | Hard — без него нет персистентности флагов | `set_flag()`, `get_flag()`, `reputation[]` |
| **Dialogue System** | Hard — без диалогов нельзя принять/сдать квест | читает `QuestSystem.get_status()` |
| **PlayerData / SaveSystem** | Hard — персистентность прогресса квестов | `active_quests[]`, `completed_quests[]` |
| **Combat / Enemy AI** | Hard — нужен для охотничьих и сюжетных (убить врагов) | `QuestSystem.notify_kill()` |
| **Skill Tree** | Soft — нужен для наград классовых квестов | `ClassAbilitySystem.unlock_ability()` |
| **Inventory / Shop** | Soft — репутация влияет на цены | `WorldState.reputation[]` |
| **Notice Board** (новый) | Soft — UI для охотничьих; без него охотничьи недоступны | генерация пула при старте сессии |

## Tuning Knobs

| Knob | Текущее | Безопасный диапазон | Эффект при изменении |
|------|---------|--------------------|--------------------|
| `MAX_ACTIVE_QUESTS` | 3 | 2–5 | Нагрузка на игрока; ниже = фокус, выше = хаос |
| `MAX_BOUNTY_ACTIVE` | 2 | 1–4 | Кол-во охотничьих одновременно |
| `BOUNTY_POOL_SIZE` | 4 | 3–8 | Разнообразие охотников за сессию |
| `base_reward` (по типу) | 50–400 | ×0.5–×2.0 от текущих | Экономический баланс |
| `reputation_thresholds` | [2, 5, 9] | [1–3, 3–7, 6–12] | Скорость роста репутации |
| `stage_multiplier` | 1.0 / 1.5 / 2.0 | 1.0–3.0 | Ощущение прогресса внутри цепочки |

## Acceptance Criteria

1. Игрок разговаривает со Старейшиной → доступен сюжетный квест; принятие → квест появляется в Quest Log со статусом ACTIVE
2. Убийство 5 врагов с активным охотничьим квестом → статус меняется на STAGE_READY; без активного квеста — счётчик не растёт
3. Возврат к NPC в STAGE_READY → диалог предлагает награду; после принятия — золото зачислено, следующая стадия ACTIVE
4. Завершённый квест → NPC больше не предлагает его; диалог изменился навсегда
5. Классовый квест завершён → способность разблокирована в ClassAbilitySystem (проверяется через HUD умений)
6. 3 квеста активны → попытка взять 4-й → отказ с пояснением в диалоге
7. Выход с активным охотничьим квестом → при следующей сессии охотничий сброшен; сюжетный/классовый/репутационный — сохранён
8. Репутация RESPECTED → скидка 10% у связанного NPC-торговца (проверяется в магазине)

## Open Questions

- **Notice Board**: где стоит доска в городе? Нужен арт и сцена
- **NPC класса**: 3 отдельных NPC (по одному на класс) или один NPC меняет диалог в зависимости от класса?
- **Охотничьи при смерти в данже**: квест сбрасывается или продолжается? (текущий дизайн: сбрасывается — требует плейтест)
- **Репутация**: глобальная или per-location? (текущий дизайн: per-location → `WorldState.reputation[loc_id]`)
- **Отказ от охотничьего**: бесплатно или штраф репутации?
