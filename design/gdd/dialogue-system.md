# Диалоговая система

> **Status**: In Design
> **Author**: User + Claude
> **Last Updated**: 2026-04-16
> **Implements Pillar**: Pillar 2 (Живой мир)

## Overview

Диалоговая система — движок ветвящихся разговоров с NPC. При приближении к NPC появляется подсказка [E]; нажатие открывает DialogueBox с текстом и вариантами ответов. Каждый NPC хранит своё дерево диалогов в отдельном JSON-файле (`assets/data/dialogues/[npc_id].json`). `DialogueSystem.gd` (Autoload) загружает нужный файл, вычисляет условия веток (флаги WorldState, статус квестов, класс игрока), отображает актуальный узел и выполняет действия по итогу ветки (принять квест, открыть магазин, выдать награду). Система не хранит игрового состояния — только читает из WorldState и QuestSystem.

## Player Fantasy

Нажал E — NPC смотрит на тебя и говорит что-то конкретное. Не «здравствуй, герой» — а «ты вернулся, я слышал про тварей в третьем зале». После выполненного квеста диалог другой. После роста репутации — другой снова. Диалог — это зеркало того, что ты уже сделал в мире.

## Detailed Design

### Core Rules

**Активация:**
1. При входе игрока в радиус NPC (50px) → над NPC появляется иконка [E]
2. Нажатие E → `DialogueSystem.start(npc_id)` — загружает `assets/data/dialogues/[npc_id].json`, вычисляет стартовый узел, открывает DialogueBox
3. Диалог блокирует управление игроком (движение, атаку) на время разговора
4. Выход из диалога: кнопка ESC или выбор варианта без продолжения

**Структура диалогового дерева (JSON):**
```json
{
  "npc_id": "elder",
  "nodes": {
    "start": {
      "conditions": [
        { "flag": "quest_elder_ch1_done", "value": true, "next": "after_quest" },
        { "quest_status": "quest_elder_ch1", "value": "STAGE_READY", "next": "complete_quest" }
      ],
      "default": "intro",
      "speaker": "elder",
      "text": "...",
      "choices": [
        { "text": "Расскажи подробнее", "next": "details" },
        { "text": "Принять задание", "next": "accept", "action": "offer_quest:quest_elder_ch1" },
        { "text": "До свидания", "next": null }
      ]
    }
  }
}
```
5. Каждый узел содержит: `speaker` (npc_id или "player"), `text`, `choices[]`, `conditions[]`, `actions[]`
6. Условия вычисляются сверху вниз; первое совпавшее определяет узел; `default` — если ни одно не совпало
7. Действия (`actions`) выполняются при выборе варианта: `offer_quest`, `complete_quest`, `open_shop`, `set_flag`, `give_gold`
8. Варианты выбора (`choices`) ведут к следующему узлу или закрывают диалог (`"next": null`)

**Условия веток:**

| Тип условия | Пример | Источник |
|-------------|--------|---------|
| `flag` | `"quest_elder_done": true` | `WorldState.get_flag()` |
| `quest_status` | `"quest_id": "ACTIVE"` | `QuestSystem.get_status()` |
| `player_class` | `"class": "warrior"` | `PlayerData.player_class` |
| `reputation` | `"location": "town", "min": 3` | `WorldState.get_reputation()` |
| `level` | `"min_level": 3` | `PlayerData.level` |

### States and Transitions

```
IDLE          →  PROMPT     : игрок вошёл в радиус NPC
PROMPT        →  IDLE       : игрок вышел из радиуса (иконка [E] исчезает)
PROMPT        →  ACTIVE     : игрок нажал E
ACTIVE        →  CHOICE     : узел NPC отображён; ожидание выбора игрока
CHOICE        →  PLAYER     : игрок выбрал вариант → портрет игрока + его реплика
PLAYER        →  ACTIVE     : переход к следующему узлу NPC
ACTIVE        →  ACTION     : выбранный вариант содержит action
ACTION        →  ACTIVE     : action выполнен → следующий узел
ACTION        →  CLOSED     : action = "close" или next = null
ACTIVE        →  CLOSED     : ESC / конечный узел
CLOSED        →  IDLE       : DialogueBox закрыт; управление возвращено игроку
```

### Interactions with Other Systems

| Система | Направление | Интерфейс |
|---------|-------------|-----------|
| **World State** | upstream → | `get_flag()`, `get_reputation()` — условия веток; `set_flag()` — как action |
| **Quest System** | upstream → | `get_status(quest_id)` — условия; `offer_quest()`, `complete_quest_stage()` — actions |
| **PlayerData** | upstream → | `player_class`, `level` — условия веток; портрет зависит от класса |
| **Shop** | downstream → | action `open_shop` → `ShopUI.open(npc_id)` |
| **NPC System** | upstream → | NPC вызывает `DialogueSystem.start(npc_id)` при взаимодействии |
| **Asset Pipeline** | upstream → | JSON-файлы в `assets/data/dialogues/`; портреты в `assets/art/characters/portraits/` |

## Formulas

Диалоговая система не содержит математических формул. Технические параметры:

- **Радиус активации**: 50px от центра NPC
- **Скорость typewriter**: 40 символов/сек (настраивается)
- **Cross-fade портрета**: 0.1–0.15 сек
- **Размер JSON диалога**: ~2–10 КБ на NPC; загружается один раз, кешируется в памяти

## Edge Cases

1. Игрок входит в радиус двух NPC одновременно → [E] показывается у ближайшего; диалог открывается с ним
2. JSON-файл диалога отсутствует → `DialogueSystem` логирует ошибку, показывает fallback «[NPC молчит]»; игра не крашит
3. Условие ссылается на несуществующий флаг → `WorldState.get_flag()` возвращает `false`; переходит к `default`
4. Действие `offer_quest` для уже активного квеста → `QuestSystem` возвращает `false`; диалог показывает узел «задание уже взято»
5. Игрок нажимает ESC во время выполнения action → action завершается, затем диалог закрывается
6. Атака врага во время диалога → диалог принудительно закрывается; управление возвращается немедленно
7. Портрет NPC отсутствует в assets → показывается placeholder-силуэт; нет краша
8. Портрет игрока для класса не найден → показывается дефолтный портрет игрока

## Dependencies

| Система | Тип зависимости | Интерфейс |
|---------|----------------|-----------|
| **World State** | Hard — флаги и репутация нужны для веток | `get_flag()`, `get_reputation()`, `set_flag()` |
| **Quest System** | Hard — статус квестов для веток и actions | `get_status()`, `offer_quest()`, `complete_quest_stage()` |
| **PlayerData** | Hard — класс и уровень для условий; класс для портрета | `player_class`, `level` |
| **NPC System** | Hard — NPC инициирует диалог | `DialogueSystem.start(npc_id)` |
| **Shop** | Soft — action open_shop | `ShopUI.open(npc_id)` |
| **Asset Pipeline** | Hard — JSON и портреты должны существовать | файловая система |

## Tuning Knobs

| Knob | Текущее | Диапазон | Эффект |
|------|---------|---------|--------|
| Радиус активации [E] | 50px | 30–100px | Больше = легче активировать |
| Скорость typewriter | 40 символов/сек | 20–instant | Темп чтения |
| Cross-fade портрета | 0.15с | 0–0.3с | 0 = мгновенная смена |
| Макс. вариантов выбора | 4 | 2–6 | Больше = сложнее выбирать |
| Затемнение фона | 50% | 30–80% | Фокус на диалог vs видимость мира |

## Visual / UI Requirements

**Общая композиция:**
- Диалог занимает нижние ~40% экрана; верхние 60% — затемнённый игровой мир
- Полупрозрачный тёмный фон под всей диалоговой панелью

**Портрет говорящего:**
- Большое изображение персонажа: ~300×400px, слева от текстового блока
- NPC говорит → портрет NPC (бюст или полная фигура, PNG с прозрачным фоном)
- Игрок выбирает реплику → портрет игрока (класс-зависимый: warrior / guardian / paladin)
- Переключение: cross-fade 0.15с между портретами
- Говорящий — полная яркость; молчащий — затемнён (alpha 0.4) или скрыт

**Текстовый блок:**
- Имя говорящего над репликой, выделено цветом (каждый NPC — свой цвет имени; игрок — нейтральный)
- Реплика NPC: typewriter-эффект; Space/клик → показать всё мгновенно
- Реплика игрока (после выбора): показывается кратко (1–2 сек или до клика) перед переходом к ответу NPC

**Варианты ответа:**
- Появляются после завершения typewriter'а
- Список 2–4 вариантов; навигация мышью или ↑↓ + Enter
- Иконки: «!» = принять квест, «✓» = сдать квест, монета = открыть магазин

**Поток кадра:**
```
NPC говорит (портрет NPC + текст)
  → typewriter завершён → появляются варианты ответа
  → игрок выбирает → портрет игрока + реплика игрока
  → cross-fade → портрет NPC + следующая реплика NPC
```

**Арт-зависимости:**
- Портрет каждого NPC: 1 выражение в MVP; 2–3 (нейтральное/довольное/недовольное) в Vertical Slice
- Портрет игрока: 3 варианта по классу
- Формат: PNG с прозрачным фоном, 300×400px

## Acceptance Criteria

1. Вход в радиус NPC → иконка [E] над ним; выход → исчезает
2. Нажатие E → DialogueBox открывается; портрет NPC слева; движение игрока заблокировано
3. Typewriter-эффект текста; Space → мгновенный показ
4. Выбор варианта → портрет меняется на портрет игрока + краткая реплика → затем снова портрет NPC
5. Условие по флагу: диалог Старейшины **до** квеста → узел intro; **после** завершения → узел after_quest
6. Action `offer_quest` → квест появляется в Quest Log; NPC продолжает диалог
7. Action `open_shop` → ShopUI открывается; DialogueBox закрывается
8. Отсутствующий JSON → fallback «[NPC молчит]»; нет краша
9. ESC → диалог закрыт; управление возвращено
10. Атака врага во время диалога → диалог принудительно закрыт

## Open Questions

- **Локализация**: тексты в JSON — отдельные файлы на язык или ключи → таблица переводов?
- **Озвучка**: в MVP нет; в Vertical Slice — звук «печатания» или голосовые реплики?
- **История диалогов**: нужен ли флаг «seen» для пропуска анимации у уже виденных реплик?
- **Выражения портрета**: как NPC-узел указывает какое выражение использовать? Поле `portrait_variant: "happy"` в JSON?
