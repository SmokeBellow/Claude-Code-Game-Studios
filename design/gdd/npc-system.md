# Система НПС

> **Status**: In Design
> **Author**: User + Claude
> **Last Updated**: 2026-04-16
> **Implements Pillar**: Pillar 2 (Живой мир)

## Overview

Система НПС определяет всех неигровых персонажей города: их роли, расположение, поведение и интерфейсы. В MVP город содержит 8 объектов взаимодействия: Старейшина (сюжетные квесты), Кузнец (снаряжение), Алхимик (расходники), Волшебник (сброс навыков), три NPC класса — Наставник воина / Страж-капитан / Паладин-жрец (классовые квесты, доступны с уровня 3), и Доска объявлений (охотничьи квесты). Каждый NPC — сцена в Godot с компонентами: `NPCBody2D` (физика, визуал, радиус [E]) + `DialogueComponent` (ссылка на JSON) + `RoleComponent` (логика роли). Все квест-гиверы бессмертны (`immortal = true`).

## Player Fantasy

Город — не просто хаб между данжами. Это место с лицами. Старейшина стоит у ратуши и ждёт твоего возвращения. Кузнец звенит молотом, алхимик возится с колбами. После выбора класса появляется новый человек которого раньше не было — или был, но теперь видит тебя иначе. Каждый NPC — точка связи между тем что ты сделал и тем что будет дальше.

## Detailed Design

### Core Rules

**Реестр NPC (MVP):**

| ID | Имя | Роль | Расположение | Immortal | Доступность |
|----|-----|------|-------------|---------|-------------|
| `elder` | Старейшина | Сюжетные квесты | У ратуши (центр) | ✅ | Всегда |
| `blacksmith` | Кузнец | Магазин (снаряжение) | Кузница (север) | ✅ | Всегда |
| `alchemist` | Алхимик | Магазин (расходники) | Лавка (восток) | ✅ | Всегда |
| `wizard` | Волшебник | Сброс дерева навыков | Башня (запад) | ✅ | Всегда |
| `warrior_mentor` | Наставник воина | Классовые квесты | Арена (юг) | ✅ | Только если class = warrior |
| `guardian_mentor` | Страж-капитан | Классовые квесты | Казармы (юг) | ✅ | Только если class = guardian |
| `paladin_mentor` | Паладин-жрец | Классовые квесты | Храм (север) | ✅ | Только если class = paladin |
| `notice_board` | Доска объявлений | Охотничьи квесты | Центральная площадь | N/A | Всегда |

**Архитектура NPC-сцены:**
1. Каждый NPC — отдельная сцена (`res://src/town/npcs/[npc_id].tscn`)
2. Компоненты: `CharacterBody2D` (коллизия, спрайт, анимация) + `Area2D` (радиус 50px для [E]) + `DialogueComponent` (ссылка: `assets/data/dialogues/[npc_id].json`)
3. При входе игрока в `Area2D` → `DialogueComponent` показывает иконку [E]
4. Нажатие E → `DialogueSystem.start(npc_id)`
5. `immortal = true` → NPC не имеет HP, не реагирует на атаки, не может быть убит

**Доска объявлений — особый случай:**
6. Это не NPC, а `Interactable2D` нода; взаимодействие через E так же
7. При входе в новую сессию → `NoticeBoard.generate_stock()` выбирает 4 случайных охотничьих квеста из пула
8. Отображает список квестов в отдельном UI (не через DialogueSystem)

**Видимость NPC класса:**
9. Все три NPC класса физически присутствуют в городе всегда, но `Area2D` активен только для NPC выбранного класса игрока
10. Если класс не выбран (`player_class = NONE`) → ни один NPC класса не реагирует на [E]

### States and Transitions

**Состояния NPC:**

| Состояние | Условие | Поведение |
|-----------|---------|-----------|
| `IDLE` | Нет игрока рядом | Idle-анимация; иконка [E] скрыта |
| `AVAILABLE` | Игрок в радиусе 50px | Иконка [E] видна |
| `TALKING` | Диалог открыт | Анимация «смотрит на игрока»; [E] скрыта |
| `LOCKED` | NPC класса, класс не выбран или чужой | `Area2D` отключён; иконка не появляется |

**Переходы:**
```
IDLE → AVAILABLE      : игрок вошёл в Area2D
AVAILABLE → IDLE      : игрок вышел из Area2D
AVAILABLE → TALKING   : игрок нажал E → DialogueSystem.start()
TALKING → IDLE        : диалог закрыт (ESC / конечный узел)
IDLE → LOCKED         : player_class не совпадает с ролью NPC класса
LOCKED → IDLE         : player_class выбран и совпадает
```

### Interactions with Other Systems

| Система | Направление | Интерфейс |
|---------|-------------|-----------|
| **Dialogue System** | downstream → | `DialogueSystem.start(npc_id)` — NPC инициирует диалог |
| **Quest System** | downstream → | через Dialogue actions: `offer_quest()`, `complete_quest_stage()` |
| **Shop (Кузнец/Алхимик)** | downstream → | через Dialogue action `open_shop` → `ShopUI.open(npc_id)` |
| **Skill Tree (Волшебник)** | downstream → | через Dialogue action `reset_skill_tree` → `SkillTree.reset()` |
| **World State** | upstream → | `get_flag()` — состояние диалога NPC; `get_reputation()` — смена реплик |
| **PlayerData** | upstream → | `player_class` — видимость NPC класса; `level` — условия квестов |
| **Notice Board** | — | отдельный `Interactable2D`; вызывает `BountyUI.open()` |

## Formulas

Система НПС не содержит математических формул. Технические параметры:

- **Радиус Area2D**: 50px (совпадает с Dialogue System и Loot System)
- **Кол-во NPC в MVP**: 7 полноценных NPC + 1 Notice Board = 8 объектов взаимодействия
- **JSON-файлов диалогов**: по одному на NPC = 7 файлов

## Edge Cases

1. Игрок выбрал класс → NPC класса мгновенно переходит из `LOCKED` в `IDLE` (реакция на сигнал `PlayerData.class_selected`)
2. Два NPC в радиусе одновременно → активируется ближайший (обрабатывается в Dialogue System)
3. Волшебник: сброс при 0 потраченных очков → `SkillTree.reset()` возвращает `false`; диалог: «Тебе нечего сбрасывать»
4. Кузнец: ассортимент пуст (всё куплено) → диалог открывается; ShopUI показывает «Нет в наличии»; NPC не исчезает
5. Notice Board: все 4 квеста взяты → «Заданий нет; возвращайся позже»
6. Атака по NPC (`immortal = true`) → урон не наносится; NPC не реагирует анимацией
7. Загрузка сохранения → все NPC на своих местах; диалог отражает текущий статус квестов через WorldState флаги

## Dependencies

| Система | Тип зависимости | Интерфейс |
|---------|----------------|-----------|
| **Dialogue System** | Hard — без диалогов NPC немой | `DialogueSystem.start(npc_id)` |
| **World State** | Hard — состояние диалогов и репутация | `get_flag()`, `get_reputation()` |
| **PlayerData** | Hard — класс для видимости NPC; уровень для условий | `player_class`, `level` |
| **Quest System** | Soft — NPC инициирует квесты через диалог | через Dialogue actions |
| **Shop** | Soft — Кузнец и Алхимик открывают ShopUI | `ShopUI.open(npc_id)` |
| **Skill Tree** | Soft — Волшебник сбрасывает дерево | `SkillTree.reset()` |

## Tuning Knobs

| Knob | Текущее | Диапазон | Эффект |
|------|---------|---------|--------|
| Радиус Area2D | 50px | 30–100px | Зона взаимодействия с NPC |
| Кол-во NPC в городе | 8 | 5–15 | Насыщенность города |
| Стоимость сброса (Волшебник) | не определена | 0–500 золота | Экономическая цена перераспределения |
| Idle-анимация | простая (2–4 кадра) | static–rich | Ощущение жизни города |

## Acceptance Criteria

1. Вход в радиус Старейшины → иконка [E] над ним; нажатие E → диалог открывается
2. NPC класса при `player_class = NONE` → [E] не появляется; после выбора класса → активен только нужный NPC
3. Кузнец: диалог → «Посмотреть товары» → ShopUI открывается с Equipment
4. Алхимик: диалог → ShopUI открывается с Consumables
5. Волшебник: диалог → «Сбросить навыки» → `SkillTree.reset()` вызван; очки возвращены
6. Атака по NPC → урон не наносится; NPC продолжает idle-анимацию
7. Notice Board: нажатие E → BountyUI открывается с 4 охотничьими квестами
8. Загрузка сохранения → все NPC на своих местах; диалог Старейшины отражает текущий статус квестов
