# Лут-система (Loot System)

> **Status**: Approved
> **Author**: User + Claude Code Game Studios
> **Last Updated**: 2026-03-25
> **Implements Pillar**: Рост героя (Pillar 1), Понятная глубина (Pillar 5)

## Overview

Лут-система управляет генерацией и подбором предметов в игровом мире. При гибели врага система бросает кубик против таблицы дропа этого типа врага и при успехе создаёт предмет-подбиральщик (`LootPickup`) в точке гибели. Боссы и сундуки используют гарантированные таблицы без броска «выпало / не выпало». Игрок подбирает предмет автоматически при входе в радиус подбора (Hades-стиль: минимум фрикции). Золото выпадает отдельно от экипировки — всегда, с обычных врагов и сундуков. Рерольная механика и выбор между предметами не входят в MVP. Система читает определения предметов из Item Database (#3) по `item_id` и передаёт результат подбора в Инвентарь (#17).

## Player Fantasy

Убил элитного врага — сердце ёкает: выпал оранжевый предмет. Подходишь — автоподбор, инвентарь мигает «новый!». Открываешь: "Cloak of the Fallen Knight — +12 AGI, +9% уклонения". Это лучше того, что сейчас надето. Надеваешь — статы растут, герой ощущается мощнее. Конечно, следующие пять врагов не дали ничего интересного — но именно поэтому этот эпик ощутился наградой, а не ширпотребом. Сундук в конце комнаты — маленькое обещание: там точно что-то есть. Босс — большая награда, заслуженная. Лут не делает игрока всемогущим сразу — он делает следующие 10 минут чуть лучше прежних 10 минут.

## Detailed Design

### Core Rules

**Источники лута**

1. **Враги**: при гибели бросают кубик (drop_chance) против своей таблицы дропа. Один враг — максимум 1 предмет-экипировка/расходник за бой. Золото выпадает всегда, независимо от броска на предмет.
2. **Сундуки**: гарантированно содержат 1–2 предмета (определяется типом сундука) + фиксированное золото. Нет броска «выпало / не выпало».
3. **Боссы**: гарантированно дропают 2 предмета (1× из Rare/Epic таблицы + 1× из elevated таблицы данжа) + большой пакет золота.
4. **Магазин (#22)**: покупка за золото — отдельная система, Лут-система предоставляет только данные о доступных предметах.

**Шансы и редкости по типу источника**

| Источник | Шанс дропа предмета | Common | Uncommon | Rare | Epic |
|----------|---------------------|--------|----------|------|------|
| Melee / Ranged | 30% | 70% | 25% | 5% | 0% |
| Elite | 60% | 40% | 40% | 17% | 3% |
| Boss drop_1 (guaranteed) | 100% | 0% | 0% | 50% | 50% |
| Boss drop_2 (elevated) | 100% | 0% | 20% | 50% | 30% |
| Сундук обычный | 100% (1 предмет) | 40% | 35% | 20% | 5% |
| Сундук боссовый | 100% (2 предмета) | 0% | 30% | 45% | 25% |

**Таблицы дропа: структура (тип + данж)**

5. Каждый враг имеет поле `loot_table_id: String` в `EnemyData.tres` (напр. `"melee_dungeon1"`). Сигнал `enemy_died` расширен полем `enemy_data: EnemyData` — Лут-система читает `loot_table_id` из него напрямую.
6. `LootTable.tres` — ресурс с **отдельными пулами по редкости**: `common_pool`, `uncommon_pool`, `rare_pool`, `epic_pool` — каждый `Array[String]` (список `item_id`). Также: `gold_min: int`, `gold_max: int`, `drop_chance: float`.
7. Бросок редкости выбирает пул напрямую по процентной таблице источника, затем равномерный random внутри пула. Рерольная механика в MVP не используется.

**Золото**

8. Золото выпадает отдельным `GoldPickup` при каждой гибели врага/открытии сундука. Мгновенный автоподбор при входе в радиус (80px).

| Источник | Золото |
|----------|--------|
| Melee | 3–8 |
| Ranged | 4–10 |
| Elite | 15–30 |
| Boss | 50–100 |
| Сундук обычный | 20–50 |
| Сундук боссовый | 80–150 |

**Авто-подбор**

9. `LootPickup` — узел `Area2D` с `CollisionShape2D` (радиус 50px). При входе игрока активируется магнит: предмет движется к игроку со скоростью 300 px/s. При касании центра игрока — подобран.
10. `LootPickup` существует максимум **300 секунд** (5 минут), затем исчезает — предотвращает накопление объектов в открытых данжах.
11. Если инвентарь полон — предмет не подбирается, показывается уведомление «Инвентарь заполнен». Предмет остаётся в мире до таймаута.

### States and Transitions

| Состояние | Условие входа | Условие выхода |
|-----------|--------------|----------------|
| **SPAWNED** | Враг умер / сундук открыт | Игрок вошёл в радиус 50px → MAGNETIZING; 300с истекло → EXPIRED |
| **MAGNETIZING** | Игрок в радиусе 50px | Предмет достиг центра игрока → COLLECTED; игрок умер → SPAWNED; игрок вышел из радиуса → SPAWNED |
| **COLLECTED** | Предмет касается центра игрока | Узел удаляется из сцены, эмитируется `item_picked_up` |
| **EXPIRED** | Прошло 300с без подбора | Узел удаляется из сцены тихо |

### Interactions with Other Systems

| Система | Направление | Данные |
|---------|-------------|--------|
| **Enemy AI (#11)** | → Loot System | Сигнал `enemy_died(xp_reward)` — триггер генерации лута; тип врага определяет `loot_table_id` |
| **Item Database (#3)** | ← Loot System | Читает определение предмета по `item_id` из таблицы дропа |
| **Инвентарь (#17)** | → Loot System | Лут-система эмитирует `item_picked_up(item_resource: ItemResource)` и `gold_picked_up(amount: int)` |
| **Audio Manager (#6)** | → Loot System | Запрос звука подбора при `COLLECTED` (разные SFX по редкости: Common/Uncommon/Rare+Epic) |

## Formulas

```
# Бросок на выпадение предмета
roll = randf()                       # [0.0, 1.0)
item_drops = (roll < drop_chance)    # drop_chance из LootTable.tres

# Выбор редкости (первый бросок — по процентам таблицы источника)
rarity_roll = randf()
# Пример для Melee: Common[0, 0.70), Uncommon[0.70, 0.95), Rare[0.95, 1.0), Epic — пул пуст

# Выбор предмета внутри редкости (второй бросок — равномерный по пулу)
# Структура Вариант B: LootTable имеет common_pool, uncommon_pool, rare_pool, epic_pool
pool = loot_table.get_pool(selected_rarity)  # возвращает Array[String] (item_id)
if pool.is_empty():
    return null  # пул пуст для данной редкости у этого источника
selected_item_id = pool[randi() % pool.size()]

# Количество золота
gold_amount = randi_range(gold_min, gold_max)   # из LootTable.tres

# Боссовый гарантированный дроп (2 предмета):
#   drop_1: выбор из объединённого rare_pool + epic_pool босс-таблицы (50% Rare / 50% Epic)
#   drop_2: рarity_roll по elevated таблице (Uncommon 20%, Rare 50%, Epic 30%),
#           затем равномерный random внутри выбранного пула той же таблицы
```

## Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| `item_id` из таблицы дропа не найден в Item Database | `load()` возвращает `null`; предмет пропускается, логируется ошибка; враг не дропает ничего |
| `item_pool` пустой или все предметы имеют `weight = 0` | Только золото выпадает; предмет не генерируется; логируется предупреждение |
| Инвентарь полон при подборе | `LootPickup` остаётся в мире, показывается уведомление «Инвентарь заполнен»; повторная попытка при следующем входе в зону |
| Игрок умирает пока `LootPickup` в состоянии MAGNETIZING | Предмет возвращается в SPAWNED; лут **не теряется при смерти** *(согласно game-concept)* |
| Два `LootPickup` перекрываются в одной точке | Оба независимы; подбираются последовательно при входе в зону; нет слияния |
| Boss дропает 2 предмета и инвентарь полон для обоих | Оба остаются в мире как SPAWNED; игрок может освободить место и вернуться |
| `LootTable.tres` с `drop_chance > 1.0` | Зажим: `clamp(drop_chance, 0.0, 1.0)` при загрузке ресурса; логируется предупреждение |
| Сундук в комнате, которую игрок уже прошёл (перезаход) | Сундук остаётся открытым — состояние сохраняется через Состояние мира (#7) *(Provisional: зависит от #7)* |

## Dependencies

**Входящие зависимости (от кого зависит эта система):**

| Система | Что берёт | Тип зависимости |
|---------|-----------|-----------------|
| **Item Database (#3)** | Определение предмета по `item_id` для создания `LootPickup` | Жёсткая |
| **Enemy AI (#11)** | Сигнал `enemy_died(xp_reward, enemy_data: EnemyData)` — триггер; Лут-система читает `enemy_data.loot_table_id` | Жёсткая |
| **Состояние мира (#7)** | Отслеживание открытых сундуков между перезаходами в комнату | Мягкая *(Provisional: зависит от дизайна #7)* |

**Исходящие зависимости (кто зависит от этой системы):**

| Система | Что берёт | Тип зависимости |
|---------|-----------|-----------------|
| **Инвентарь (#17)** | Сигналы `item_picked_up(item_resource)` и `gold_picked_up(amount)` | Жёсткая |
| **Audio Manager (#6)** | Запрос SFX подбора при переходе в COLLECTED (разные SFX по редкости) | Мягкая |
| **Магазин (#22)** | Предметы для продажи берёт через Item Database, не напрямую из Лут-системы | Косвенная |

**Интерфейс:**

```gdscript
# Сигналы, эмитируемые Лут-системой:
signal item_picked_up(item_resource: ItemResource)
signal gold_picked_up(amount: int)

# Сигнал Enemy AI (расширен для Лут-системы):
signal enemy_died(xp_reward: int, enemy_data: EnemyData)

# LootTable.tres структура (Вариант B — отдельные пулы по редкости):
class_name LootTable extends Resource
@export var drop_chance: float = 0.3
@export var common_pool: Array[String] = []    # списки item_id по редкости
@export var uncommon_pool: Array[String] = []
@export var rare_pool: Array[String] = []
@export var epic_pool: Array[String] = []
@export var gold_min: int = 3
@export var gold_max: int = 8
```

## Tuning Knobs

| Параметр | Текущее значение | Безопасный диапазон | Что ломается при выходе |
|----------|-----------------|---------------------|------------------------|
| `drop_chance` Melee/Ranged | 30% | 15%–50% | <15%: данж ощущается пустым; >50%: инвентарь переполняется быстро, редкость теряет вес |
| `drop_chance` Elite | 60% | 40%–80% | <40%: элита не ощущается особенной; >80%: чрезмерно часто |
| Rare% в дропе обычного врага | 5% | 2%–10% | >10%: Rare перестаёт ощущаться редким |
| Epic% в дропе с босса | 30% | 15%–50% | <15%: боссы разочаровывают; >50%: эпики девальвируются |
| Радиус авто-подбора | 50px | 30px–80px | <30px: раздражающий пиксельхантинг; >80px: подбирает нежелательное |
| Скорость магнита | 300px/s | 200–500px/s | <200: вялый магнит; >500: предмет телепортируется |
| Таймаут `LootPickup` | 300с | 60с–∞ | <60с: игрок не успевает вернуться; ∞: накопление объектов в сцене |
| Золото Melee | 3–8 | 1–20 | Уточняется при дизайне Магазина (#22) |
| Золото Boss | 50–100 | 30–300 | Уточняется при дизайне Магазина (#22) |

## Visual/Audio Requirements

**Визуальные требования:**
- `LootPickup` спрайт — иконка предмета из Item Database с цветовой рамкой по редкости: Common (серая), Uncommon (зелёная), Rare (синяя), Epic (оранжевая)
- Состояние SPAWNED: лёгкое покачивание/боббинг (синусоидальное, амплитуда ~3px, период ~1с)
- Состояние MAGNETIZING: предмет движется к игроку, рамка становится ярче
- Эффект при COLLECTED: вспышка-поглощение (scale → 0 за 0.2с) + частицы искр цветом редкости
- Золото: монетка-спрайт без рамки, мгновенно исчезает при подборе

**Аудио-требования:**
- SFX подбора Common/Uncommon: тихий «клик» + монетный звон
- SFX подбора Rare: более отчётливый звуковой акцент
- SFX подбора Epic: заметный отдельный звук — «момент» (не слишком громкий)
- SFX подбора золота: короткий монетный звон (общий для всех количеств)
- Звук открытия сундука: «лязг крышки» + нарастание перед дропом

## Acceptance Criteria

| # | Критерий | Как проверить |
|---|----------|---------------|
| 1 | Melee враг дропает предмет в ~30% случаев | Убить 100 Melee врагов, посчитать дропы — 20–40 предметов |
| 2 | Предмет соответствует редкостной таблице источника | Записать 50 дропов с Melee: ~35 Common, ~12 Uncommon, ~3 Rare, 0 Epic |
| 3 | `LootPickup` движется к игроку при входе в радиус 50px | Подойти к предмету — он притягивается |
| 4 | Предмет подбирается автоматически без нажатия кнопки | Зайти в радиус — предмет исчезает, попадает в инвентарь |
| 5 | Золото всегда выпадает при гибели врага | Убить 10 врагов — у каждого появляется `GoldPickup` |
| 6 | Предмет остаётся в мире при полном инвентаре | Заполнить инвентарь, подойти к луту — предмет не подбирается, уведомление |
| 7 | `LootPickup` исчезает через 300с без подбора | Оставить предмет на 5 минут — узел удаляется без ошибок |
| 8 | Босс всегда дропает ровно 2 предмета | Убить 5 боссов — каждый дал 2 предмета |
| 9 | Несуществующий `item_id` в таблице дропа не крашит игру | Добавить невалидный `item_id` в `.tres` — ошибка в логе, нет краша, предмет пропускается |
| 10 | Лут не теряется при смерти игрока | Умереть пока предмет в состоянии MAGNETIZING — предмет остаётся на земле после возрождения |
