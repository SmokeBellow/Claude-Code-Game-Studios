# Система уровней и XP (Level & XP System)

> **Status**: Approved
> **Author**: User + Claude Code Game Studios
> **Last Updated**: 2026-03-25
> **Implements Pillar**: Рост героя (Pillar 1), Понятная глубина (Pillar 5)

## Overview

Система уровней и XP отслеживает опыт героя и управляет переходами между уровнями: каждый убитый враг, завершённый квест и побеждённый босс приносят XP, и при достижении порога `current_xp >= xp_to_next_level` система испускает сигнал `level_up(attribute_points)` для Health & Stats. Уровень — это валюта прокачки: он открывает 5 очков атрибутов для распределения и, через зависимые системы, новые ветки дерева навыков. Кривая XP плавно растёт — каждый уровень требует немного больше опыта, чем предыдущий, обеспечивая ощущение прогресса без гринда. Без этой системы атрибуты остаются на уровне 1 навсегда — нет роста, нет дерева навыков, нет смысла в прокачке.

## Player Fantasy

«Дин-дин» — звук повышения уровня в правильной RPG звучит как маленькая победа. В «Хрониках» повышение уровня — это пауза посреди данжа: мир замирает, игрок смотрит на 5 очков и думает «куда вложить?». Это момент самовыражения, а не обязательный ритуал. Воин, который сложил все очки в Силу и Выносливость, ощущает как его персонаж «тяжелеет» — враги умирают за меньше ударов, он сам держит больше. Через 10 уровней это уже не тот герой, что был в начале. Прогресс — конкретный, ощущаемый, заработанный в бою. Анти-гринд: игрок никогда не должен останавливаться и убивать врагов ради XP — уровни приходят сами как следствие прохождения данжа.

## Detailed Design

### Core Rules

**Накопление XP**

1. Система хранит `current_xp: int` и `current_level: int` (стартует с 1).
2. Источники XP:
   - **Враг убит**: враг испускает `enemy_died(xp_reward: int)` при переходе в DEAD (отдельно от `died`); значение `xp_reward` берётся из `EnemyData.tres` → Melee = 10 XP, Ranged = 12 XP, Elite = 35 XP
   - **Квест завершён**: Квест-система вызывает `add_xp(amount)` напрямую (30–80 XP, зависит от квеста)
   - **Босс убит**: Boss испускает `enemy_died(xp_reward)` с повышенным значением (150 XP)
3. XP накапливается без ограничений в рамках текущего уровня; **переполнение переносится** на следующий.
4. XP никогда не опускается ниже `xp_floor` текущего уровня (граница входа в уровень). При смерти: `current_xp = xp_floor(current_level)` — теряется прогресс к следующему уровню, но не сам уровень.

**Повышение уровня**

5. При каждом добавлении XP: если `current_xp >= xp_to_next_level(current_level)` → `level_up()`.
6. `level_up()`:
   - `current_level += 1`
   - Если `current_level > MAX_LEVEL (30)` → ничего (XP более не накапливается)
   - Испускает `level_up(current_level, attribute_points=5)` → слушает Health & Stats
   - Испускает `xp_updated(current_xp, xp_to_next_level)` → обновляет XP-бар HUD

**Отображение прогресса**

7. Каждое получение XP испускает `xp_gained(amount, source_type)` — HUD показывает флоатинг-текст `+N XP`.

---

### States and Transitions

| Состояние | Условие входа | Условие выхода | Эффект |
|-----------|--------------|----------------|--------|
| **Накопление** | Старт / после level_up | `current_xp >= xp_to_next_level` | Нормальная работа, XP накапливается |
| **Level Up** | `current_xp >= xp_to_next_level` | Сигнал `level_up` испущен | `current_level++`, испускает сигнал, H&S ставит паузу для экрана атрибутов |
| **Макс уровень** | `current_level = 30` | — (финальное) | XP не накапливается далее |
| **Смерть** | Сигнал `died` от игрока | Перезагрузка сцены | `current_xp = xp_floor(current_level)` — сброс к порогу уровня |

---

### Interactions with Other Systems

| Система | Направление | Данные |
|---------|-------------|--------|
| **Health & Stats** | → Level XP | Слушает `level_up(level, points)` → паузирует игру, открывает экран атрибутов; слушает сигнал `died` для death penalty |
| **Enemy AI (#11)** | → Level XP | Каждый враг испускает `enemy_died(xp_reward: int)` при переходе в DEAD; Level XP подписывается и вызывает `add_xp(xp_reward)` |
| **Система квестов (#21)** | → Level XP | Вызывает `add_xp(amount)` при завершении квеста |
| **HUD** | ← Level XP | Слушает `xp_gained(amount, source)` (флоатинг), `xp_updated(current, max)` (XP-бар), `level_up(level, points)` (уведомление) |
| **Система классов (#15)** | ← Level XP | Слушает `level_up(level)` → проверяет разблокировку новых веток дерева навыков |
| **Сохранение/загрузка (#24)** | ← Level XP | Сохраняет `current_xp`, `current_level` как часть состояния героя |

## Formulas

```
# Требование XP для перехода с уровня n на n+1
xp_to_next_level(n) = 50 + 10 * n

# Примеры:
# n=1 (1→2): 60 XP
# n=5 (5→6): 100 XP
# n=10 (10→11): 150 XP
# n=20 (20→21): 250 XP
# n=29 (29→30): 340 XP

# Накопленный XP для достижения уровня L (от старта)
cumulative_xp(L) = (L - 1) * (50 + 5 * L)

# Граница текущего уровня (floor для death penalty)
xp_floor(current_level) = cumulative_xp(current_level)

# XP награда за тип врага (фиксировано)
xp_reward(enemy_type):
  Melee  → 10 XP
  Ranged → 12 XP
  Elite  → 35 XP
  Boss   → 150 XP

# XP за квест (задаётся в данных квеста)
quest_xp = 30–80 XP
```

**Таблица прогрессии:**

| Уровень | XP для этого уровня | Накопленный XP | Заходов в данж |
|---------|--------------------|--------------:|---------------:|
| 1 → 2   | 60                 | 60             | 0.09           |
| 5 → 6   | 100                | 400            | 0.60           |
| 7 → 8   | 120                | 630            | 0.95 ← конец 1-го захода |
| 10 → 11 | 150                | 1050           | 1.58           |
| 15 → 16 | 200                | 1875           | 2.82           |
| 20 → 21 | 250                | 3000           | 4.50           |
| 25 → 26 | 300                | 4375           | 6.57           |
| 29 → 30 | 340                | **5800**       | **8.71**       |

*1 заход = ~666 XP (30 врагов смешанного типа + 1 босс + 1 квест)*

**Параметры формулы:**

| Параметр | Значение | Tuning knob |
|----------|----------|-------------|
| `BASE` | 50 | Влияет на стоимость первых уровней |
| `INCREMENT` | 10 | Влияет на крутизну кривой |
| `MAX_LEVEL` | 30 | Хард кап (из H&S GDD) |

## Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| Игрок на MAX_LEVEL (30) убивает врагов | `add_xp()` игнорируется — XP не накапливается, флоатинг не показывается |
| Несколько level_up за один раз (большой XP-бонус) | level_up обрабатывается в loop пока `current_xp >= xp_to_next_level && current_level < MAX_LEVEL`; все сигналы испускаются; H&S суммирует очки и открывает один экран атрибутов |
| Death penalty на уровне 1 (xp_floor = 0) | `current_xp = 0` — наказание не уходит в минус |
| Смерть при 0 прогресса в уровне (только что повысился) | `current_xp = xp_floor(current_level)` = текущее значение → изменений нет |
| Враг убит и игрок умер в одном кадре | XP добавляется (`enemy_died` раньше), затем death penalty применяется. Если XP не вызвал level_up — он теряется |
| Квест завершён при MAX_LEVEL | `add_xp()` игнорируется — квест всё равно считается завершённым |
| `xp_reward` врага = 0 (настройка дизайнера) | `add_xp(0)` — никаких изменений, сигнал `xp_gained` не испускается |

## Dependencies

**Входящие (Level XP зависит от):**

| Система | Что берёт | Тип |
|---------|-----------|-----|
| **Health & Stats** | Слушает сигнал `died` для death penalty; H&S принимает `level_up(level, points)` и открывает экран атрибутов | Жёсткая |
| **Enemy AI (#11)** | Каждый враг испускает `enemy_died(xp_reward)` — Level XP подписывается и вызывает `add_xp(amount)` | Жёсткая |

**Исходящие (зависят от Level XP):**

| Система | Что берёт |
|---------|-----------|
| **Health & Stats** | Сигнал `level_up(current_level, attribute_points=5)` → распределение атрибутов |
| **Система классов (#15)** | Сигнал `level_up(current_level)` → разблокировка веток дерева навыков |
| **HUD (#25)** | Сигналы `xp_gained(amount, source)`, `xp_updated(current, max)`, `level_up(level, points)` |
| **Сохранение/загрузка (#24)** | Хранит `current_xp`, `current_level` |
| **Система достижений (#30)** | Слушает `level_up(level)` для достижений типа «Достичь уровня N» |

## Tuning Knobs

| Параметр | Текущее | Безопасный диапазон | Что ломается при выходе |
|----------|---------|---------------------|------------------------|
| `BASE` формулы | 50 | 30–80 | <30: первый уровень мгновенный, нет ощущения награды; >80: первый данж почти не даёт уровней |
| `INCREMENT` формулы | 10 | 5–20 | <5: кривая почти линейная, нет разницы между ранней и поздней игрой; >20: late-game требует чрезмерного гринда |
| XP Melee | 10 | 7–15 | Изменение смещает соотношение источников XP |
| XP Ranged | 12 | 8–18 | Должен оставаться выше Melee — иначе нет стимула преследовать Ranged |
| XP Elite | 35 | 25–50 | <25: Elite не ощущается особенным; >50: спавн Elite-комнат позволяет быстро выгрузить уровни |
| XP Boss | 150 | 100–250 | <100: босс не ощущается вехой прогресса; >250: фарм босс-комнаты нарушает анти-гринд |
| Quest XP | 30–80 | 20–120 | >120: квесты начинают конкурировать с боем как основной источник XP |

## Visual/Audio Requirements

**Визуальные эффекты:**
- **XP флоатинг**: `+N XP` всплывает над убитым врагом (маленький шрифт, жёлтый/золотой цвет), анимация подъёма + fade-out за 0.8с
- **Level Up**: яркая вспышка вокруг персонажа + анимированный текст «LEVEL UP!» в центре экрана (0.5с, затем экран атрибутов)
- **XP-бар заполнен**: короткий pulse-эффект на иконке уровня в HUD перед переходом

**Звук:**
- **+XP**: тихий мелодичный звук «тик» при каждом получении XP (слышен, не раздражает)
- **Level Up**: торжественный аккорд (2 ноты) + звук уровня (классический RPG «динг»)
- **Несколько level_up подряд**: звуки воспроизводятся с задержкой 0.1с каждый (не сливаются)

## UI Requirements

**XP-бар (HUD):**
- Тонкая полоска под HP/мана барами: текущий XP / XP до следующего уровня
- Слева: иконка уровня (число) — обновляется при level_up
- Анимация заполнения при получении XP (плавное нарастание)
- При level_up: бар мгновенно сбрасывается к новому уровню, затем заполняется overflow XP

**Экран атрибутов (при level_up):**
- Открывается поверх игры с паузой (реализуется в Health & Stats)
- Показывает: текущий уровень, количество очков для распределения, 6 атрибутов с текущими значениями и прогнозируемым эффектом (`+5 STR → +5 физ. урон`)
- XP-флоатинг отключается во время паузы

## Acceptance Criteria

| # | Критерий | Как проверить |
|---|----------|--------------|
| 1 | Melee враг даёт 10 XP при смерти | Убить Melee, проверить `current_xp` увеличился ровно на 10 |
| 2 | Level up при достижении порога — сигнал `level_up(level, 5)` испущен | Добавить XP до порога через отладчик, проверить сигнал и значения |
| 3 | H&S получает level_up → игра паузируется, экран атрибутов открывается | Повысить уровень в бою — игра замирает, экран доступен |
| 4 | Death penalty: current_xp = xp_floor(current_level) после смерти | Набрать 50% XP к уровню, умереть, проверить `current_xp == xp_floor` |
| 5 | На MAX_LEVEL (30) убийство врагов не меняет XP | Достичь уровня 30, убить врага — `current_xp` не изменился |
| 6 | Переполнение через несколько уровней корректно | Добавить 1000 XP на уровне 1 — игрок достигает level 10 (cumulative_xp(10)=900 ≤ 1000 < 1050=cumulative_xp(11)), с 100 XP прогресса в level 10→11 |
| 7 | `cumulative_xp(8) = (8-1)*(50+5*8) = 7*90 = 630` | Проверить расчёт формулы в unit-тесте |
| 8 | Переполнение при нескольких level_up: все сигналы испущены | Добавить XP для 3 уровней сразу — получить 3 сигнала `level_up`, 15 суммарных очков |

## Open Questions

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | XP за квест: фиксированное или зависит от сложности? | MVP: фиксированное по квесту (задаётся в данных квеста, 30–80 XP) |
| 2 | Масштабируется ли XP за врагов с уровнем данжа? | MVP: нет — фиксировано; Vertical Slice: рассмотреть scaling |
| 3 | Показывать ли XP-флоатинг для каждого врага отдельно? | Да — каждый враг показывает своё значение независимо |
| 4 | Death penalty: только XP или также лут? | Только XP — лут не теряется (из game-concept: «умереть обидно, но не разрушительно») |
| 5 | Нужны ли XP-бонусы за стиль (без урона, скорость комнаты)? | Не в MVP; рассмотреть как бонус опциональный в Vertical Slice |
