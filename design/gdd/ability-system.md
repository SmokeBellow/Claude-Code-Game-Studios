# Система способностей (Ability System)

> **Status**: Approved
> **Author**: User + Claude Code Game Studios
> **Last Updated**: 2026-03-25
> **Implements Pillar**: Рост героя (Pillar 1), Умная победа (Pillar 3)

## Overview

Система способностей — единый менеджер всех активных способностей героя: хранит определения (базовый урон, стоимость маны, cooldown, тип эффекта, радиус), управляет состоянием cooldown и проверяет доступность перед активацией. Система является посредником между боевым вводом (Combat System) и ресурсами персонажа (Health & Stats): Combat вызывает `activate_ability(id)`, а Ability System самостоятельно проверяет ману, списывает её через Health & Stats и запускает cooldown. Без этой системы невозможны ни масштабирование способностей через Аркану, ни будущее Дерево навыков, разблокирующее новые способности.

## Player Fantasy

Способности — это не «кнопка больше урона», это инструменты с характером. Вихрь Воина создан для толпы — бессмысленно тратить его на одного врага. Будущие способности будут такими же: каждая отвечает на конкретный вопрос «что делать в этой ситуации?». Разблокировать новую способность — значит получить новый ответ на старый вопрос. Игрок, у которого есть три способности, видит арену иначе, чем тот, у кого одна. Система способностей — это «словарный запас» героя: чем он богаче, тем элегантнее решения.

## Detailed Design

### Core Rules

**Структура способности**

1. Каждая способность — `AbilityResource` (.tres) с полями: `ability_id`, `display_name`, `icon`, `ability_type` (AoE / Targeted / Buff / Projectile), `base_damage`, `mana_cost`, `cooldown`, `radius` (для AoE), `range` (для Targeted/Projectile), `buff_stat` / `buff_value` / `buff_duration` (для Buff/Debuff).
2. Герой имеет **2 слота способностей**: `slot_E` и `slot_Q`. Способности назначаются при создании персонажа (MVP: фиксированы по классу).

**Способности Воина (MVP)**

- **E — «Вихрь»**: тип AoE, радиус 60px, 7 тиков по 40% урона за 2с, +30% скорость, стоимость 35 маны, cooldown 8с *(полная спецификация в Combat System GDD)*
- **Q — «Рывок»**: тип Targeted, Воин совершает анимированный прыжок (0.2с полёт) к ближайшему врагу в радиусе 200px и наносит ×1.5 урона при приземлении. Во время полёта движение и все действия заблокированы (нет i-фреймов). После приземления: 0.5с cooldown на ПКМ (парирование) — предотвращает мгновенный Q→ПКМ→LMB комбо. Стоимость: 20 маны. Cooldown: 5с.

**Жизненный цикл активации**

3. Combat вызывает `activate_ability(slot: String)` → Ability System:
   - a. Проверяет: способность назначена в слот
   - b. Проверяет: `current_mana >= mana_cost` (через Health & Stats)
   - c. Проверяет: cooldown истёк (`remaining_cooldown <= 0`)
   - d. Если всё ок → списывает ману (`spend_mana`), запускает cooldown, испускает сигнал `ability_activated(ability_id, caster_position, facing_direction)`
   - e. Если нет → испускает сигнал `ability_failed(slot, reason)` для HUD
4. Урон способности рассчитывается: `ability_damage = base_damage * ability_power` (Аркана из Health & Stats).
5. Cooldown тикает в `_process` каждый кадр, независимо от боевого состояния.

### States and Transitions

| Состояние слота | Условие входа | Условие выхода | Что заблокировано |
|-----------------|--------------|----------------|-------------------|
| **Ready** | Старт / cooldown истёк | Успешная активация | — |
| **Active** | `activate_ability` успешна | Эффект завершён (длительность / одиночный удар) | Повторная активация этого слота |
| **Cooldown** | Эффект завершён | `remaining_cooldown <= 0` | Активация |
| **Unavailable** | Нет маны / слот пуст | Мана восстановлена / способность назначена | Активация |

*Два слота (E и Q) работают независимо — cooldown одного не влияет на другой.*

### Interactions with Other Systems

| Система | Направление | Данные |
|---------|-------------|--------|
| **Health & Stats** | ← Ability System | Читает `current_mana`, `max_mana`, `ability_power` (Аркана) |
| **Health & Stats** | → Ability System | Вызывает `spend_mana(amount)` при успешной активации |
| **Combat System** | → Ability System | Вызывает `activate_ability(slot)` при нажатии E/Q |
| **Combat System** | ← Ability System | Слушает `ability_activated(id, pos, dir)` для запуска эффекта |
| **Item Database** | ← Ability System | Читает предметы с бонусами `ABILITY_POWER` и `MAX_MANA` *(через Health & Stats)* |
| **HUD** | ← Ability System | Слушает: `ability_failed(slot, reason)`, `cooldown_updated(slot, remaining)` для отображения статуса |
| **Дерево навыков** | → Ability System | Вызывает `assign_ability(slot, ability_id)` при разблокировке новой способности |

## Formulas

```
# Урон способности (магический тип)
ability_damage = base_damage * ability_power
# ability_power = 0.83 + 0.034 * effective(ARC)  ← из Health & Stats GDD
# На ур.1 (ARC=5): ability_power = 1.0
# При ARC=55 (SC2): ability_power = 2.36

# Рывок — урон при приземлении (физический тип, не ability_power)
rush_damage = (weapon_base + phys_damage_bonus) * 1.5

# Вихрь — урон за тик (7 тиков за 2с)
whirl_tick = (weapon_base + phys_damage_bonus) * 0.4   # ← из Combat System GDD

# Cooldown тик
remaining_cooldown -= delta  # каждый _process кадр
ability_ready = remaining_cooldown <= 0

# Проверка доступности
can_activate = (slot != null) AND (current_mana >= mana_cost) AND ability_ready
```

| Переменная | Источник | Значение на ур.1 |
|------------|----------|-----------------|
| `ability_power` | Health & Stats (ARC) | 1.0× |
| `current_mana` | Health & Stats | 50 |
| `weapon_base` | Item Database | ~25 |
| `phys_damage_bonus` | Health & Stats (STR) | +5 |

*Рывок использует физический урон (STR), а не ability_power (ARC) — Воин силовой класс, не маг.*

## Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| E и Q нажаты одновременно | Обрабатываются по порядку ввода в одном кадре — оба проходят проверку независимо |
| Мана кончилась в середине Вихря | Вихрь не прерывается — мана списана при активации, не тикует во время эффекта |
| Рывок — нет врагов в радиусе 200px | Способность не активируется, мана не списывается, cooldown не запускается |
| Рывок — враг умер в момент прыжка | Воин приземляется на позицию врага, урон не наносится (цели нет) |
| Рывок — стена между Воином и целью | Воин прыгает, огибает препятствие по move_and_slide, останавливается перед стеной; урон наносится по области перед стеной если враг достижим |
| Рывок — парирование сразу после приземления | Заблокировано на 0.5с — предотвращает Q→ПКМ→LMB комбо |
| `assign_ability` вызван во время Active | Назначение откладывается до перехода слота в Cooldown/Ready |
| Способность активирована при HP = 1 | Активация проходит — проверяется только мана, не HP |
| Cooldown сбросился через предмет/навык (будущее) | `remaining_cooldown = 0` → переход в Ready в следующем кадре |
| `ability_id` не найден в базе способностей | Логируется ошибка, слот переходит в Unavailable |

## Dependencies

**Входящие (Ability System зависит от):**

| Система | Что берёт | Тип |
|---------|-----------|-----|
| **Health & Stats** | `current_mana`, `ability_power`; вызывает `spend_mana(amount)` | Жёсткая |
| **Item Database** | Предметы с бонусами `ABILITY_POWER` и `MAX_MANA` *(через Health & Stats)*; `AbilityResource` (.tres) хранятся отдельно в `assets/data/abilities/`, не в Item Database | Мягкая |

**Исходящие (зависят от Ability System):**

| Система | Что берёт |
|---------|-----------|
| **Combat System** | Сигналы `ability_activated`, `ability_failed`; метод `activate_ability(slot)` |
| **HUD** | Сигналы `cooldown_updated(slot, remaining)`, `ability_failed(slot, reason)` |
| **Дерево навыков** | Метод `assign_ability(slot, ability_id)` |
| **Система загадок** | Сигнал `ability_activated` — некоторые загадки реагируют на конкретные способности |

## Tuning Knobs

| Параметр | Текущее значение | Безопасный диапазон | Что ломается при выходе |
|----------|-----------------|---------------------|-------------------------|
| Стоимость маны Вихря (E) | 35 | 20–50 | <20: спам без ресурс-менеджмента; >50: нельзя использовать на ур.1 |
| Cooldown Вихря (E) | 8с | 5–15с | <5с: Вихрь становится основной атакой; >15с: игрок забывает о способности |
| Стоимость маны Рывка (Q) | 20 | 10–35 | <10: Рывок бесплатный по ощущению; >35: конкурирует с Вихрем за ману |
| Cooldown Рывка (Q) | 5с | 3–10с | <3с: Рывок заменяет позиционирование движением; >10с: игрок игнорирует способность |
| Дальность Рывка | 200px | 120–300px | <120px: слишком коротко для разрыва дистанции; >300px: телепорт через всю комнату |
| Множитель урона Рывка | ×1.5 | ×1.0–×2.5 | <×1.0: не стоит использовать; >×2.5: Рывок лучше обычной атаки |
| Количество слотов способностей | 2 (E+Q) | 1–4 | <2: мало тактики; >4: перегрузка интерфейса и баланса в MVP |

## Visual/Audio Requirements

**Визуальные эффекты:**
- **Вихрь** — частицы вращения + подсветка радиуса 60px (полупрозрачный круг)
- **Рывок** — след за Воином во время прыжка + вспышка при приземлении
- **Cooldown** — иконка слота затемняется, заполняется как pie-chart по мере восстановления
- **Нет маны** — иконка слота мигает синим 2–3 раза

**Звук:**
- **Вихрь активирован** — свист раскручивающегося оружия *(из Combat System GDD)*
- **Рывок** — короткий звук прыжка + тяжёлый удар при приземлении
- **Ability failed (нет маны)** — глухой отказ, тихий

## UI Requirements

**Панель способностей (HUD):**
- Два слота под HP/мана барами: `[Q]` и `[E]`
- Каждый слот: иконка 48×48px + клавиша + cooldown таймер (секунды)
- **Ready**: яркая иконка
- **Cooldown**: затемнённая иконка + pie-chart анимация + цифра оставшихся секунд
- **Unavailable (нет маны)**: синяя рамка мигает

## Acceptance Criteria

| # | Критерий | Как проверить |
|---|----------|---------------|
| 1 | E активирует Вихрь при наличии маны — мана списывается, cooldown запускается | Нажать E, проверить ману и cooldown в отладчике |
| 2 | Q активирует Рывок — Воин прыгает к ближайшему врагу в 200px и наносит ×1.5 урона | Встать рядом с врагом, нажать Q, проверить урон и позицию |
| 3 | При нехватке маны — способность не активируется, HUD показывает причину | Истратить ману, нажать E/Q — нет эффекта, есть индикатор в HUD |
| 4 | Во время cooldown — повторная активация игнорируется | Нажать E сразу после Вихря — ничего не происходит |
| 5 | Два слота cooldown независимы — E на cooldown не блокирует Q | Активировать E, сразу нажать Q — Рывок срабатывает |
| 6 | Рывок при отсутствии врагов в радиусе не тратит ману и не запускает cooldown | Нажать Q в пустой комнате — мана и cooldown не изменились |
| 7 | `ability_power` из Арканы масштабирует урон способностей | Поднять ARC до 30, проверить урон vs ARC=5 — урон ~1.85× больше |
| 8 | `assign_ability` назначает способность в слот — активируется после назначения | Вызвать `assign_ability("slot_Q", id)`, нажать Q — новая способность срабатывает |
| 9 | Сигнал `cooldown_updated` корректно обновляет HUD-таймер | Активировать E, наблюдать таймер 8→0 в HUD |
