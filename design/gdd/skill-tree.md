# Skill Tree / Система классов

> **Status**: In Design
> **Author**: User + Claude
> **Last Updated**: 2026-04-15
> **Implements Pillar**: Pillar 1 (Рост героя), Pillar 5 (Понятная глубина)

## Overview

Skill Tree — система прокачки, в которой игрок развивает героя через два слоя: общие узлы (доступны всем классам) и ветки специализации (3 ветки на класс × 5 узлов). Игрок получает 1 очко навыка при старте и по 1 очку за каждый уровень начиная со 2-го. Общие узлы открывают специализации, узлы специализации разблокируют активные умения в слотах R/F/G. Доступ — через вкладку «Навыки» в SidePanel (Tab) в любой момент, плюс автооткрытие после level-up (после экрана атрибутов).

## Player Fantasy

Каждое вложенное очко делает героя заметно другим: сначала — ощущение роста силы (общие узлы), затем — ощущение специализации, когда герой становится Берсерком, Стражем или Паладином. К финалу данжа игрок управляет персонажем, который играется принципиально иначе чем в начале — не просто сильнее, а иначе.

## Detailed Design

### Core Rules

1. Игрок получает 1 очко навыка при старте, +1 за каждый уровень начиная со 2-го
2. Ветка `general` доступна всегда; содержит 2 узла-шлюза перед специализациями
3. Узлы внутри ветки открываются строго последовательно — нельзя перепрыгнуть узел
4. Класс выбирается на 3-м уровне (ClassSelectionScreen); до выбора доступна только `general`
5. После выбора класса открываются 3 ветки специализации, но очки вкладываются только в **одну** — выбор фиксируется при первом вложении очка ⚠️ *impl gap: не реализовано в SkillTree.gd*
6. Пассивные узлы применяются к StatsComponent немедленно; активные узлы проверяются через `is_node_unlocked()` в ClassAbilitySystem
7. Слоты умений разблокируются по кол-ву очков в выбранной специализации: Q(слот 0) ≥ 1, E(слот 1) ≥ 3, F(слот 2) ≥ 5
8. Сброс дерева доступен в городе (через NPC или меню) — возвращает все очки, снимает все пассивные эффекты ⚠️ *impl gap: не реализовано*

### States and Transitions

**Состояния ветки:**

| Состояние | Условие входа | Визуал |
|-----------|--------------|--------|
| `LOCKED` | Класс не выбран; или `general < 2`; или ветка чужого класса | Серый, недоступен |
| `AVAILABLE` | `general ≥ 2`, класс выбран, ветка не заблокирована, другая ветка не выбрана | Подсвечена, можно вложить |
| `ACTIVE` | Первое очко вложено в эту ветку | Активна, остальные spec-ветки — `LOCKED` |
| `FULL` | Все 5 узлов куплены | Completed-состояние, очки нельзя потратить здесь |

**Переходы:**
```
LOCKED → AVAILABLE  : general куплен (2 узла), класс выбран
AVAILABLE → ACTIVE  : первый spend_point() в эту ветку
ACTIVE → FULL       : 5-й узел куплен
ACTIVE → AVAILABLE  : сброс дерева в городе (все ветки возвращаются в AVAILABLE)
FULL → AVAILABLE    : сброс дерева в городе
```

**Балансовое ограничение**: Суммарное количество очков за игровую сессию (уровень 1 → max практический уровень) должно точно совпадать с общей ёмкостью дерева: 2 (general) + 5 (spec) = 7. ⚠️ *Требует настройки скорости выдачи очков — см. Tuning Knobs*

### Interactions with Other Systems

| Система | Направление | Интерфейс |
|---------|-------------|-----------|
| **Level/XP System** | upstream → | Сигнал `level_up(level, pts)` → `+1 skill_point` |
| **StatsComponent** | → downstream | Пассивы: `apply_skill_passive(passive_id)` при покупке и при `_restore_passives()` |
| **ClassAbilitySystem** | → downstream | Активные узлы: `is_node_unlocked(branch, index)` определяет доступность умений (Q=0, E=1, F=2) |
| **ClassSelectionScreen** | → downstream | Выбор класса на уровне 3 задаёт `PlayerData.player_class` → разблокирует ветки |
| **PlayerData / SaveSystem** | ↔ bidirectional | `skill_points` и `spent_points` персистируются, `_restore_passives()` восстанавливает эффекты |
| **Wizard NPC** (новый) | → downstream | Сброс: возврат всех очков + снятие пассивов ⚠️ *не реализован* |

## Formulas

- **Очки навыков при уровне N**: `skill_points_total = N` (1 стартовое + 1 за каждый уровень ≥ 2)
- **Ёмкость дерева**: `capacity = 2 (general) + 5 (spec) = 7 узлов`
- **Балансовое равенство**: практический max уровень за сессию ≈ 7. Если игрок растёт быстрее — снизить частоту выдачи очков (например, каждые 2 уровня)

## Edge Cases

1. Тратить очко в spec без выбора класса → `spend_point()` возвращает `false`, очко сохраняется
2. Тратить очко когда ветка заполнена (5/5) → rejected, очко сохраняется до сброса
3. Смерть → дерево не сбрасывается; только явный сброс у Волшебника в городе
4. Попытка сброса вне города → UI-кнопка/NPC недоступны за пределами города
5. Два `spend_point()` в одном кадре → очки тратятся последовательно, нет race condition

## Dependencies

| Система | Тип зависимости |
|---------|----------------|
| Level/XP System | Hard — источник skill points |
| StatsComponent | Hard — применяет пассивные эффекты |
| ClassAbilitySystem | Hard — читает состояние дерева для активных умений |
| ClassSelectionScreen | Hard — без выбора класса ветки недоступны |
| PlayerData / SaveSystem | Hard — персистентность состояния дерева |
| Wizard NPC | Soft — сброс, улучшает UX но не критичен для базового геймплея |

## Tuning Knobs

| Knob | Текущее | Безопасный диапазон | Эффект при изменении |
|------|---------|--------------------|--------------------|
| `GENERAL_GATE` | 2 | 1–3 | Насколько быстро открываются специализации |
| Частота выдачи очков | каждый уровень | каждый 1–2 уровня | Скорость заполнения дерева |
| Стоимость сброса у Волшебника | не определена | 0–500 золота | Экономическая цена перераспределения |
| Бонусы пассивных узлов | см. `NODES` в SkillTree.gd | ±50% от текущих | Относительная сила каждого узла |

## Acceptance Criteria

1. Игрок достигает уровня 3 → автоматически появляется ClassSelectionScreen
2. После выбора класса в SidePanel (Навыки) → 3 ветки специализации отображаются и доступны
3. Первое очко в spec-ветку → остальные 2 ветки этого класса становятся недоступны (серые, заблокированы)
4. Пассивный узел куплен → StatsComponent немедленно отражает изменение (проверяется через HUD атрибутов)
5. 1 / 3 / 5 очков в spec-ветке → слоты Q / E / F разблокируются в HUD
6. Сброс у Волшебника → все очки возвращены, StatsComponent сброшен до базовых значений, spec-ветки снова доступны
