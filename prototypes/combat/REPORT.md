# Prototype Report: Combat System

**Date**: 2026-03-23
**Status**: Code complete — awaiting manual playtest in Godot 4.6

---

## Hypothesis

Реальновременный топ-даун бой с атаками, уклонением (i-фреймами) и одной
способностью (nova burst) будет ощущаться отзывчивым и тактически интересным.
Враги с простым AI (patrol → chase → attack) создадут достаточное напряжение
для проверки гипотезы «умная победа».

---

## Approach

Построен минимальный играбельный бой за одну сессию:

- **Player.gd**: движение WASD, атака ЛКМ (hitbox через Area2D), dodge Space
  (i-фреймы 220ms + 50ms после), nova ability ПКМ (AOE урон в радиусе 90px)
- **Enemy.gd**: конечный автомат PATROL → CHASE → ATTACK, aggro при ударе
- **DamageNumber.gd**: всплывающие числа урона
- **CombatPrototype.gd**: главная сцена, UI, kill counter, рестарт

**Упрощения (намеренные):**
- Нет анимаций — только flash-модуляция цвета
- Нет звука
- Нет карты — открытое пространство
- Hitbox — мгновенный (1 physics frame), не анимированный замах
- AI без pathfinding — прямолинейное преследование

## Метрики для проверки (заполнить после плейтеста)

### Ощущение движения
- [ ] Отзывчивость: задержка между вводом и движением < 1 кадра?
- [ ] Скорость 180px/s: ощущается как бодрое / медленное / быстрое?

### Ощущение атаки
- [ ] Hitbox честный: враги умирают когда ты в них «попадаешь»?
- [ ] Attack cooldown 0.4s: достаточно быстро, не ощущается как ограничение?
- [ ] Flash на враге читается как попадание?

### Уклонение
- [ ] I-фреймы 220ms: достаточно для уклонения от атаки врага?
- [ ] Dodge cooldown 0.8s: правильный ритм — не слишком часто/редко?
- [ ] Ощущение: эскейп-механика или просто быстрое движение?

### Ability
- [ ] Nova burst удовлетворяет при сметании 2+ врагов?
- [ ] Cooldown 3s ощущается правильно?

### AI
- [ ] Враги замечают игрока в нужный момент (не слишком рано/поздно)?
- [ ] Атака врага 12 урона / 1.2s: угрожает, но не мгновенно убивает?

## Ключевые вопросы для плейтеста

1. **Тактика vs закликивание**: возникает ли желание думать о позиции,
   или игрок просто стоит и атакует?
2. **Уклонение**: используется ли dodge как тактический инструмент,
   или только как паника-кнопка?
3. **Ability**: ощущается ли nova как значимое решение или просто «нажать
   когда готово»?
4. **Читаемость**: понятно ли что происходит без анимаций?

---

## Известные ограничения прототипа

- **Нет pathfinding**: враги застревают в углах. В продакшне нужен
  NavigationAgent2D.
- **Hitbox — мгновенный**: в продакшне нужен анимированный замах с
  чётким окном попадания.
- **Нет звука**: audio feedback критически важен для ощущения удара —
  прототип не проверяет это.
- **Простой AI**: не проверяет «умную победу» — для этого нужны враги
  с паттернами (Фаза 2 прототипа).

## Рекомендация

### Предварительная: PROCEED с условиями

На основе архитектуры кода и анализа механик:

Базовые системы (движение, hitbox, i-фреймы, AI) архитектурно корректны
для Godot 4.6. Паттерны (CharacterBody2D + move_and_slide, Area2D hitboxes,
конечный автомат врага) — стандартные и хорошо задокументированные.

**Однако**: финальная рекомендация PROCEED/PIVOT зависит от плейтеста.
Главный риск — бой ощущается как закликивание без тактики. Это нельзя
определить без игры.

### После плейтеста: что проверить

**Если PROCEED:**
- Продакшн реализация требует: анимации (AnimationPlayer), звуковой фидбек,
  NavigationAgent2D для AI, отдельные компоненты (HealthComponent,
  HitboxComponent) вместо встроенных скриптов
- Первая продакшн система для проектирования: **Боевая система GDD**

**Если PIVOT:**
- Вариант A: Добавить замах с окном попадания (0.2s windup → 0.2s active)
  для большей тактильности
- Вариант B: Добавить ресурс стамины для dodge/ability — создаёт
  ресурс-менеджмент
- Вариант C: Дать врагам паттерны (телеграфируемые атаки) — проверить
  тогда пиллер «умная победа»

## Уроки для других систем

1. **AI врагов** в продакшне требует NavigationAgent2D — заложить в GDD
2. **Hitbox-система** должна быть компонентом, не встроенной логикой —
   нужна для переиспользования в enemy attacks, traps, puzzles
3. **I-фреймы** — критическая механика для честного ощущения боя;
   продолжительность (220ms) требует тонкой настройки по результатам
   плейтеста
4. **Состояние мира** (World State System) должна уметь хранить kill_count
   и другие боевые события — учесть при проектировании системы
